# 연애 상담 페르소나 v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 공통 코어를 판-중립 base(톤·세이프티·화법)로 정리하고, 연애 상담을 코어 상속 하위 문서로 재작성하며, `computeTurnSignals`를 관계 chat 라우트에 이식해 심문 피로를 해소한다.

**Architecture:** 코어(byeolkong_core.md)의 판-특정 표현만 도메인-중립으로 일반화(삭제 아님) — 정체성·§3 훅 근거·§"언제" 표현. "판 읽기" 정체성은 사주/타로 하위 문서가 자체 명시(이미 대부분 있음, 한 줄 보강). 릴레이션은 `getRelationshipPersona() = getCore()+관계도메인` 그대로 상속하되 관계 도메인 파일을 전면 재작성(판 미얹음 → 누출 0). 배관은 `buildRelationshipSystemMessage`에 turnSignals 추가 + 라우트에서 주입.

**Tech Stack:** 마크다운 페르소나(`data/persona/*.md`, 런타임 `readFileSync` — 컴파일 안 됨), `lib/claude.ts`(TS), Next 16 API 라우트. 검증: `npx tsc --noEmit`(코드) + `grep`(프로즈) + dev 서버 재시작 + 3d QA 하네스(별도, 전 페르소나 회귀 포함).

**Spec:** `docs/superpowers/specs/2026-07-19-연애상담-페르소나-design.md`

⚠️ **배포 정책:** dev 전용 커밋/푸시. prod(main)은 사이클 3e 일괄 — 이 플랜 범위 밖.
⚠️ **페르소나/데이터 파일 수정 후 dev 서버 재시작 필수**(모듈 캐시 — `getCore`/`getRelationshipPersona` 가 첫 호출 결과를 캐싱).
