# 마이페이지 위계·리듬 개선 (A+C 하이브리드) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 마이페이지의 위계를 살린다 — 프로필 히어로를 띄우고, 사주 목록을 self 제외 + 일주 한자 타일 리치 행으로, 계정 메뉴를 설정형 리스트로 분리한다.

**Architecture:** 전부 `app/mypage/page.tsx`(클라이언트 컴포넌트) 내부 편집. 오행 색상 상수만 `SajuBoard.tsx`에서 `lib/saju/elements.ts`로 추출해 마이페이지와 공유(DRY). 데이터/API 변경 없음 — 이미 로드된 `profiles[].saju`(SajuResult)에서 일주 한자/일간을 읽는다.

**Tech Stack:** Next.js 16 (App Router, RSC/CSR), TypeScript, Tailwind v4. 기존 흐름 재사용: `ProfileForm` 인라인 편집, 탈퇴 확인 모달, `/api/profiles`.

> **검증 방식:** 이 레포는 단위 테스트 프레임워크가 없다. 각 태스크 검증은 `npx tsc --noEmit` + 마지막에 `npm run build` + dev 브라우저 육안 확인. 새 테스트 프레임워크 도입 안 함(기존 패턴 + YAGNI).
