# 내 고민톡 표면 통일 리디자인 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 내 고민톡(`/readings`) 리스트 카드를 마이페이지와 동일한 "흰 카드 + 부드러운 그림자"로 통일하고, 탭 활성색·리딩 타일 프레임을 정리한다.

**Architecture:** 전부 `app/readings/page.tsx`(클라이언트 컴포넌트) 내부의 className 교체. 카드 정보 구조·동작·생성중 다크 카드는 변경 없음.

**Tech Stack:** Next.js 16, TypeScript, Tailwind v4.

> **검증 방식:** 단위 테스트 프레임워크 없음. 각 태스크 `npx tsc --noEmit` + 마지막 `npm run build` + dev 브라우저 육안 확인. 새 프레임워크 도입 안 함.

## File Structure

수정: `app/readings/page.tsx` — 상담/운세 카드 + 빈상태 카드 표면, 탭 활성색, 운세 타일 프레임.

(신규 파일 없음. 생성중 다크 카드 / 헤더 / 페이지네이션 / 카드 정보 구조는 손대지 않는다.)

## Task 1: 카드 · 빈상태 · 탭 표면 통일

**Files:**
- Modify: `app/readings/page.tsx`

> 공통 shadow 토큰: `shadow-[0_2px_10px_rgba(159,138,208,0.08)]` (마이페이지 카드와 동일).

- [ ] **Step 1: 상담(consult) 카드 표면**

`app/readings/page.tsx`에서 상담 탭 카드의 `<Link>` className을 교체.

찾기 (정확히):
```
className="bg-cream-warm rounded-2xl p-3.5 border border-lilac-mid/30 flex gap-3 items-start hover:border-lilac-deep/50 transition"
```
교체:
```
className="bg-white rounded-2xl p-3.5 border border-lilac-mid/20 shadow-[0_2px_10px_rgba(159,138,208,0.08)] flex gap-3 items-start hover:border-lilac-deep/50 transition"
```

- [ ] **Step 2: 운세(fortune) 카드 표면**

운세 탭 카드의 `<Link>` className을 교체. (상담 카드와 `flex` 순서가 달라 별도임 — `flex items-start gap-3`)

찾기 (정확히):
```
className="bg-cream-warm rounded-2xl p-3.5 border border-lilac-mid/30 flex items-start gap-3 hover:border-lilac-deep/50 transition"
```
교체:
```
className="bg-white rounded-2xl p-3.5 border border-lilac-mid/20 shadow-[0_2px_10px_rgba(159,138,208,0.08)] flex items-start gap-3 hover:border-lilac-deep/50 transition"
```

- [ ] **Step 3: 빈 상태 카드 표면**

찾기 (정확히):
```
<div className="bg-cream-warm rounded-2xl p-6 border border-lilac-mid/30 text-center">
```
교체:
```
<div className="bg-white rounded-2xl p-6 border border-lilac-mid/20 shadow-[0_2px_10px_rgba(159,138,208,0.08)] text-center">
```

- [ ] **Step 4: 탭 활성색**

탭 버튼의 활성 분기 className 문자열을 교체.

교체:
```

```

## Task 2: 운세 리딩 타일 프레임 통일

**Files:**
- Modify: `app/readings/page.tsx`

배경: 상담/타로 타일은 이미 `w-12 h-12 rounded-xl`인데 운세(비-생성중) 타일만 `w-10 h-10 rounded-lg`라 제각각. 운세 타일을 동일 프레임으로 맞춘다. (생성중 다크 카드의 타일은 변경하지 않는다.)

- [ ] **Step 1: 운세 카드 타일 크기·모서리 통일**

찾기 (정확히 — `bg-gold-soft/30` 가 붙은 쪽이 비-생성중 운세 카드 타일):
```
<div className="shrink-0 self-center w-10 h-10 rounded-lg bg-gold-soft/30 flex items-center justify-center text-[18px]">
```
교체:
```
<div className="shrink-0 self-center w-12 h-12 rounded-xl bg-gold-soft/30 flex items-center justify-center text-[18px]">
```

> 주의: 생성중 카드의 타일(`bg-gold/15` 가 붙은 `w-10 h-10 rounded-lg`)은 **건드리지 않는다** — 다크 카드 유지 대상.

- [ ] **Step 2: 타입 컴파일 검증**

Run: `npx tsc --noEmit`
Expected: PASS.

- [ ] **Step 3: 커밋**

```bash

git commit -m "style(readings): 운세 리딩 타일 프레임 w-12 rounded-xl로 통일"
```
