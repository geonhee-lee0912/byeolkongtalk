# 오늘의 운세 결과 템플릿 리뉴얼 — 설계

- 날짜: 2026-06-06
- 범위: **오늘의 운세(daily)** 결과 화면 + 리포트 생성 방식. 월간(monthly)·사주(saju_full)는 이번 범위 밖(기존 마크다운 섹션 유지).
- 배경: 현재 daily 리포트는 제목 없는 줄글 한 덩어리로 들어가 읽기 어렵고 임팩트가 약함. 한 장 카드 안에서 구조화된 템플릿 + 별콩이 문체·내용 규칙을 정해 매번 일관된 결과가 나오게 한다.

---

## 1. 레이아웃 (화이트 미니멀, 단일 카드)

노션·토스 같은 깔끔한 화이트 톤. 카드 박스로 쪼개지 않고 **한 장 안에서 hairline 구분선으로만** 섹션을 나눈다.

위 → 아래 순서:

1. **상단 바** — 좌측 `오늘의 운세` 타이틀, 우측 날짜(예: `6월 6일 토요일`)
2. **일진 히어로** — 오늘 들어온 두 글자(예: `甲子`)를 **유일한 큰 컬러 포인트**(보라 `#5A3E8C`)로 크게. 아래 `갑자 · 오늘 들어온 기운` + 오행 점 두 개(예: 목 甲 / 수 子, 오행별 색 점)
3. **한 줄 총평** — 굵은 한두 줄 요약 (중앙)
4. **종합운 별점** — `오늘 종합운 ★★★★☆` 한 줄. **해시태그 칩 바로 위**에 위치. 별은 회색 톤에 채운 별만 보라로 절제
5. **칩 3종** — 고스트(테두리만) 칩: `# 키워드` / `색` / `시간대`
6. **도입 섹션 — 오늘 들어온 두 글자** (제목 텍스트, 본문 4~5문장)
7. **6개 도메인 섹션** — 각 `아이콘 + 제목` + 본문, hairline 구분선으로 구분:
   - 💰 재물운
   - 💼 직장 · 업무운
   - 💗 애정 · 관계운
   - 🌿 건강 · 멘탈운
   - 📄 학업 · 문서 · 이동운
   - ⚖️ 오늘의 균형 (`✅ 좋아` 한 줄 / `⚠️ 주의` 한 줄)
8. **별콩이의 한마디** — **다크모드 블록**(밤보라 `#211A33` 배경 + 골드 `#F5D680` 제목 + 밝은 텍스트). **좌상단 원형 아이콘에 별콩이 프로필 이미지**(`/byeolkong-main.png`) + 이름 라벨

> 시각 레퍼런스: `.superpowers/brainstorm/259-1780731604/content/white-minimal.html`

### 색 / 톤 규칙
- 배경 흰색, 섹션 구분은 얇은 라인(`#F0EEF4`)만.
- 일진 두 글자 = 유일한 큰 컬러 포인트(보라). 별점·칩은 거의 흑백으로 절제.
- 오행 점 색은 코드 매핑(목 초록 `#5FA36A`, 수 파랑 `#5B86C9` 등 — 5행 전부 정의).

## 2. 출력 계약 (구조화 JSON)

오늘의 운세는 **동기 방식 한 번 생성 → 저장 → 결과 화면이 읽어오는** 구조(채팅 스트리밍 아님)라 JSON이 적합. 템플릿의 칸(별점·칩·균형·6섹션)이 항상 채워지도록 자유 텍스트 파싱 대신 JSON으로 받는다.

### AI가 생성하는 부분
```jsonc
{
  "stars": 4,                       // 종합운 1~5 정수 (일진×일간 조화 반영)
  "summary": "새 시작의 기운이 도는 날 — 미뤄둔 한 걸음을 떼기 좋아",
  "lucky": { "keyword": "시작", "color": "라벤더", "time": "오전" },
  "intro": "오늘 들어온 두 글자 풀이 (4~5문장)",
  "sections": [
    { "key": "money",  "body": "..." },   // 제목은 코드 고정, AI는 본문만
    { "key": "work",   "body": "..." },
    { "key": "love",   "body": "..." },
    { "key": "health", "body": "..." },
    { "key": "study",  "body": "..." }
  ],
  "balance": { "good": "...", "warn": "..." },
  "note": "별콩이의 한마디 (1~2문장)"
}
```
- 도메인 제목/아이콘/순서는 **코드 고정**(AI는 `key`에 맞는 `body`만). AI가 제목을 바꾸거나 섹션을 누락하지 못하게.
- `sections`는 정확히 5개(money/work/love/health/study) + balance + intro + note.

### 서버가 병합하는 부분 (만세력 계산값 — 결정론적)
일진 두 글자·오행·색은 AI가 아니라 `calcTemporalLuck`의 계산값을 서버가 JSON에 합쳐 저장한다.
```jsonc
{
  "iljin": {
    "stem": "甲", "branch": "子", "hanja": "갑자",
    "stemElement": "목", "branchElement": "수"
  }
}
```
- 최종 저장본 = `AI JSON + iljin` 한 덩어리(문자열로 `messages.content`에 저장).
- 결과 화면은 이 저장본만 파싱해 렌더 → **공유 퍼블릭 링크도 sajuData 없이 그대로 렌더됨**(시크릿창 공유 시 빈 화면/로그인 문제 자연 해결).
- 오행 → 색 매핑은 코드 상수.

### 파싱 실패 안전망
- AI 응답이 유효 JSON이 아니거나 필수 필드 누락 시: 1회 재생성 또는 에러 처리(생성 실패 메시지). 깨진 템플릿을 그리지 않는다. (구현 시 재시도 1회 → 실패면 기존 generate 에러 흐름 재사용.)

## 3. 별콩이 문체 · 내용 규칙

기존 페르소나(`data/persona/byeolkong_fortune.md`) 보이스를 유지하고, 오늘의 운세 전용 규칙을 더한다.

### 보이스 (기존 유지)
- 반말, 다정한 또래 친구 톤.
- **단정 금지** — 운명을 정하지 않고 흐름·가능성·선택을 비춤("~할 거야" ✕ → "~한 흐름이 보여", "~해보면 좋아").
- **불안 금지** — 안 좋은 기운도 따뜻한 대비·조언으로. 겁주는 표현 ✕.
- 점술 용어·한자 나열 ✕. 사주 용어는 친구가 알아듣게 풀어서.
- 두루뭉술 ✕ — 받은 사주·일진 데이터를 근거로 구체적으로.

### 오늘의 운세 전용 규칙
- **분량**: 도입(오늘 들어온 두 글자)·6개 도메인 본문 각 **4~5문장**. 기존 페르소나의 "전체 600~900자" 규칙은 daily에 한해 덮어쓴다.
- **연결성**: 각 도메인은 오늘 일진 기운과 연결해서 풀기(랜덤 일반론 ✕).
- **균형**: 좋기만 한 예언 ✕. 도메인마다 챙길 점/주의도 자연스럽게.
- **별점**: 2~5 범위로 정직하게. 매일 4~5만 주지 않기. 단 1점 같은 겁주는 점수는 지양(불안 금지 원칙).
- **칩(키워드·색·시간)·오늘의 균형**: 일진 기운에서 끌어내기. 무작위 ✕.
- 도입은 두 글자가 어떤 기운인지 + 그 기운이 이 사람의 일간·사주와 만나 오늘 어떤 영향을 주는지를 쉽게 풀어 시작.

## 4. 구현 범위

| 파일 | 변경 |
|---|---|
| `lib/fortune/prompt.ts` | daily `SECTION_GUIDE`를 JSON 출력 지시로 교체. 도메인 키·분량·문체 규칙 명시. (monthly/saju_full 불변) |
| `app/api/fortune/create/route.ts` | daily 분기: AI JSON 파싱·검증 → 일진 계산값(`iljin`) 병합 → 저장. 실패 시 재시도/에러. |
| `app/fortune/result/page.tsx` | daily 분기를 새 템플릿 렌더로 교체. 저장본 JSON 파싱 후 필드 렌더. (비-daily 분기 불변) |
| 신규 상수 | 오행 → 색 매핑(5행). 도메인 메타(key→제목·아이콘·순서). |

### 영향 / 호환
- 기존에 저장된 (구버전 줄글) daily 리포트: JSON 파싱 실패 시 **기존 줄글 렌더로 폴백**(하위호환). 신규 생성분만 새 템플릿.
- 공유(`/api/readings/[id]/public`) 경로는 저장본을 그대로 반환하므로 추가 변경 없음.
- 공유 텍스트(`handleShare`)는 새 필드 기준으로 텍스트 조립 갱신.

## 5. 범위 밖 (YAGNI)
- 월간·사주 리포트 템플릿(이번엔 daily만).
- 별점/칩의 애니메이션, 도메인 접기/펼치기.
- 오늘의 운세 결과의 이미지(OG) 공유 — 텍스트/링크 공유 유지.
