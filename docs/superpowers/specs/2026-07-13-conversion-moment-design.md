# 전환 순간 만들기 — 설계 (페르소나 + 크로스셀 + 결과 화면 추천)

**작성일**: 2026-07-13
**근거**: [페이월 퍼널·대화 품질 findings](2026-07-13-paywall-funnel-conversation-findings.md) — 도달→결제 7.7%, 결제 시도 0건, 구매 의사를 페르소나가 꺾는 사례, 회피+[END], 시기 질문 크로스셀 부재.
**목표**: 유저의 "더 보고 싶다"는 식욕이 살아있는 순간을 꺾지 않고 다음 상담으로 잇는다. 페이월 도달→결제 전환율을 다음 코호트에서 개선 (baseline 7.7%).

## 확정된 설계 결정

|---|---|

| **크로스셀 범위** | 타로→사주(시기) + 사주→타로(상대 마음) 양방향 + **결과 화면 추천까지 이번 사이클** |

| **종료 턴 처리** | 미해결 확답 질문이 걸려있으면 **같은 턴 안에서 방향 답 → 작별 → [END]** (턴 연장 없음) |
| **유저 선택권** ★ | 대화 중간에 다른 상담을 안내할 땐 **절대 지금 넘어가라고 하지 않는다** — "이 대화 마저 풀어도 되고, 준비되면 마무리하고 넘어가도 돼". 이미 낸 별이 아깝지 않게. 넘어가는 통로는 항상 **정식 종료(그레이스풀 [END]) 경유** |
| **인챗 추천 버튼** | 별콩이가 크로스셀/새 리딩을 언급한 그 응답에 **[RECO:] 마커** → 프론트가 인라인 카드 렌더 (대화당 1회). 클릭 → 확인 모달(선택권) → **A안: 그레이스풀 종료 후 결과 화면 건너뛰고 바로 새 상담 진입** (3단계 동선) |

## 변경 ① 페르소나 — "더 보고 싶다" 신호 규칙 (신설)

`data/persona/byeolkong.md` + `byeolkong_tarot.md`에 새 섹션 **"더 보고 싶다는 신호 (반드시)"**:

- **신호**: "카드 더 뽑아줘", "새로 봐줘", "다른 것도 봐줘", "한 번 더", "~도 봐줘" 류 — 유저가 추가 리딩 욕구를 표현하는 모든 발화.
- **절대 말리지 않는다**: "오늘 카드로 충분히 들여다본 것 같아", "더 뽑는다고 명확해지는 게 아니라" 류 **금지**. 이 욕구는 별콩이를 좋아한다는 신호다 — 반겨라.
- **응답 공식**: ①욕구 반김("더 보고 싶구나, 좋아") → ②이 상담의 카드/판은 고정임을 짧게 → ③경로 안내 2갈래:
  - 같은 고민을 더 깊이 → "이 대화 끝나면 **결과 화면의 '이어가기'**로 지난 맥락 기억한 채 이어볼 수 있어"
  - 새 질문/새 카드 → "**새 상담**에서 카드를 새로 펼치면 그때 흐름으로 봐줄게"
- **선택권 문구 필수**: "지금 이 대화에서 더 풀어도 되고, 준비되면 마무리하고 넘어가도 돼" — 지금 나가라고 몰지 말 것.
- **우선순위 명시**: 이 규칙은 수렴/마무리 지시보다 우선한다. 수렴 톤이어도 구매 의사 신호에는 이 공식으로 답한 뒤 마무리를 잇는다.

## 변경 ② 페르소나 — §"언제·될까" 개정: 어순 반전 + 회피구 상한

두 파일의 기존 §"언제·될까·얼마나" 섹션 수정:

- **3단 순서 반전**: 기존 ①한계 인정→②방향 답 을 **①방향 답 먼저 → ②한계는 뒤에 짧게 → ③실행 한마디** 로. 첫 문장이 회피면 유저는 "또 거절"로 읽는다.
  - ❌ (현행 예시) "정확한 시점은 카드도 못 찍어 — … 대신 이 카드는 ~"
  - ✅ "이 카드 흐름은 ~쪽으로 열린 결이야. 확정까진 카드 밖의 일이지만, 이 흐름이면 ~할 가능성이 더 보여. 그동안 ~를 다져두면 좋아."
- **회피구 대화 전체 1회**: "~못 찍어", "~정해줄 수 없어", "~카드가 알려줄 영역이 아니야" 류 명시적 한계 문장은 **질문이 바뀌어도 한 대화에 1회만**. (현행 규칙은 "같은 질문 재요구 시"만 커버 → 결혼생각/마음/호감처럼 질문을 바꾸면 회피구가 리셋되던 구멍을 막음.) 두 번째부터는 한계 언급 없이 방향만 구체화.

## 변경 ③ 페르소나 — 크로스셀 라우팅 (양방향)

- **타로→사주**: 유저가 시기·날짜("좋은 날", "언제쯤", "몇 월 안에")를 물으면 — §언제·될까 방식으로 방향 답을 준 **뒤에** — "날짜의 결은 카드보다 **사주 '좋은 날'**(태어난 흐름으로 앞으로 30일을 짚는 상담)이 더 잘 봐. 이 대화 끝나면 결과 화면에서 이어볼 수 있어."
- **사주→타로**: 유저가 상대방 마음("그 사람 무슨 생각이야", "날 좋아할까")을 파고들면 — 사주 관점 답을 준 뒤 — "상대 마음의 결은 **타로 관계 카드**가 더 잘 비춰줘."
- **남발 방지**: 크로스셀 언급은 **대화당 1회**, 반드시 방향 답을 준 다음에만. 유저가 반응 없으면 반복하지 않는다.
- **선택권 문구 동반 필수** (결정 ★): "지금 대화 마저 풀고 가도 되고" 를 항상 곁들인다.

## 변경 ③-b 인챗 추천 버튼 ([RECO:] 마커 + 인라인 카드)

과거 결정(e9158de "대화는 별콩이↔사용자만" — 추천 질문 칩 제거)과의 정합: 그때 칩은 **매 턴 떠있는 대화 조종 UI**, 이번은 **별콩이 발화에 앵커된 1회성 전환 어포던스** — 성격이 다름.

- **마커**: 페르소나가 크로스셀(③)·새 리딩 안내(①)를 말하는 바로 그 응답에 `[RECO:<product>]` 마커 삽입 (제품 enum은 ⑤와 동일). [CARD:n] 인프라 재사용 — 화면에선 마커 제거, 프론트가 감지.
- **인라인 카드**: 그 assistant 메시지 아래 작은 카드 ("🌟 '좋은 날' 상담으로 이어보기 / 지난 고민을 기억한 채 이어져요"). 대화당 최대 1개.
- **클릭 → 확인 모달** (선택권 보장): "이 대화를 마무리하고 넘어갈까? 남은 대화가 있다면 계속해도 좋아요" — [아니, 대화 더 할래] / [마무리하고 넘어가기].
- **"대화 더 할래" 시**: 모달만 닫힘. **카드는 해당 메시지에 앵커된 채 유지** (대화가 이어지면 자연히 위로 스크롤). 재조르기 없음 — 페르소나도 재언급 금지(1회 규칙), 카드도 재생성 없음. 마음 바뀌면 스크롤 올려 다시 클릭 가능.
- **"마무리하고 넘어가기" 시 (A안 확정)**: forceEnd 그레이스풀 종료(작별 멘트 재생) → **결과 화면 건너뛰고 바로** 추천 상담 진입 (cross-type fresh, ⑤의 클릭 동선과 동일 파라미터). 3단계 동선.
  - 기존 대화는 [END]가 찍힌 **정식 완료 상태** — 내 고민톡에 남고, 결과 화면(`from=history`)·이어가기(ContinuationModal이 내 고민톡·결과 양쪽 마운트) 모두 가능. 소실되는 것 없음.

## 변경 ④ 코드 — 종료 턴 구조 강제 (`lib/claude.ts`)

- `absHardcapGuide`(절대캡·'대화 마무리' 버튼)와 `convergeLastGuide`에 추가:
  > 직전 유저 발화가 아직 답 못 받은 확답/시기 질문이면, 작별 인사 전에 **반드시 §언제·될까 방식의 방향 답부터** 준 다음 그레이스풀 마무리 + [END]. 회피 문장으로 시작해서 닫지 말 것.
- `gracefulClosingBlock`의 '이어가기' 초대 구체화: 고정 문구("이 고민, 다음에 이어가기로 다시 만나자") 대신 **이 대화의 미해결 화두를 짚어서** — "다음엔 그 사람 기류가 어떻게 변했는지 같이 보자" 식으로. (재방문 명분 부여)
- **결과 화면 예고 한 줄** ★: 대화 중 크로스셀을 언급했거나 미해결 확답 질문이 남은 채 닫힐 때, 마무리에 "**결과 화면에 '좋은 날'로 이어보는 길을 준비해뒀어** — 준비되면 거기서 이어가자" 류 예고 포함. (자연 종료 유저가 결과 카드를 만났을 때 "별콩이가 말한 그거" 연결)
- 사주/타로 두 빌더(`buildSystemMessage` 계열) 동일 적용.

## 변경 ⑤ 코드 — 결과 화면 "다음 상담 추천" (LLM 태깅)

### 태깅 (우선순위: 마커 → haiku)
- **1순위 — [RECO:] 마커**: 대화 메시지에 [RECO:] 마커가 있으면 그 상품이 추천 (haiku 호출 없음). 마무리 멘트의 "준비해뒀어"와 결과 카드가 항상 일치.
- **2순위 — haiku**: 마커 없는 대화만, chat 라우트에서 [END] 스트림 완료 시 **fire-and-forget haiku 호출** (기존 `detectSensitiveAsync` 패턴 재사용). forceEnd 종료도 [END]가 찍히므로 동일 생성.
- 입력: 대화 전문(또는 마지막 N턴) + 상담 타입/상품. 출력 JSON:
  ```json
  {
    "unresolvedQuestion": "연락이 언제 올까",     // 없으면 null
    "recommendedProduct": "saju:good_days",       // enum 밖 값은 폐기
    "hookCopy": "'연락 언제 올까' — 그 날의 결은 좋은 날 상담이 짚어줄 수 있어"
  }
  ```
- **추천 후보 enum**: `saju:good_days`(시기·날짜) / `saju:nature`(본질·방향) / `saju:choice`(선택 갈림길) / `tarot:relation`(상대 마음, 관계 스프레드) / `continue`(같은 고민 계속 — 기본값). 미해결 질문이 없으면 `continue`.
- 가드: `has_sensitive=true`면 태깅 생략(추천 없음). fortune 리포트 대상 아님. 파싱 실패·enum 밖 값이면 조용히 버림(추천 없음 = 기존 화면).

### 저장
- `readings.next_reco JSONB` (nullable) — 마이그레이션 1개. `{product, question, hook, created_at}`.

### 결과 화면
- `next_reco` 있으면 기존 이어가기 블록 위/자리에 **추천 카드**: 훅 카피 + 해당 상담 진입 버튼. 없으면 현행 그대로 (태깅 경합·실패 시 우아한 폴백).
- `product='continue'`면 기존 이어가기 모달 흐름 그대로, 훅 카피만 얹음.

### 추천 클릭 → cross-type 이어가기 (fresh)
- 추천 카드 클릭 시 **continuation fresh 모드의 cross-type 확장**으로 진입:
  - `previous_reading_id` = 이번 reading (어트리뷰션 — "추천→전환" 측정이 이 링크로 가능)
  - `continuation_mode = 'fresh'` (기존 의미와 일치: 새 판 + 부모 고민 요약 주입 + **정가**)
  - 부모 요약 주입으로 새 상담이 "지난번에 연락 시기가 궁금했었지"를 알고 시작
  - 타입 간 이동(타로 reading → 사주 상품) 허용하도록 continue 흐름 확장. 사주 상품 진입 시 프로필(생년월일)이 없으면 기존 사주 입력 흐름 경유.
- deep(할인) 모드는 cross-type 미지원 — 다른 판이므로 의미 없음. 동일 타입 이어가기는 기존 그대로.

## 흐름 종합 (유저 여정)

```
대화 중 "재회운 좋은 날 있을까?"
  → (②) 방향 답 먼저 + (③) "날짜는 사주 '좋은 날'이 더 잘 봐. 지금 대화 마저 풀고 가도 되고,
     준비되면 넘어가도 돼" + [RECO:saju:good_days] 마커       ← 선택권
  → (③-b) 그 메시지 아래 인라인 카드 "🌟 '좋은 날'로 이어보기"
  ┌ 경로 1 — 인챗 카드 클릭: 확인 모달 → [마무리하고 넘어가기]
  │   → forceEnd 그레이스풀 종료 → 결과 화면 스킵, 바로 '좋은 날' 진입 (A안, 3단계)
  │   → 기존 대화는 완료 상태로 내 고민톡에 보존 (결과·이어가기 가능)
  ├ 경로 2 — [아니, 대화 더 할래]: 카드 유지된 채 대화 계속 → 자연 수렴
  │   → (④) [END] 마무리에 "결과 화면에 이어보는 길 준비해뒀어" 예고
  │   → 결과 화면: (⑤) 추천 카드 (마커 기반, 마무리 멘트와 일치) → 클릭 → 진입
  └ 경로 3 — 크로스셀 언급 없이 끝난 대화: [END] → haiku 태깅 → 결과 화면 추천 카드
       (셋 다 도착지는 cross-type fresh 이어가기: previous_reading_id 링크 + 부모 요약 주입 + 정가)
```

## 검증 계획

1. **페르소나 회귀**: conversation-qa-harness에 시나리오 추가 —
   - 수렴 중 "카드 더 뽑을 수 있어?" → 말리지 않고 경로+선택권 안내하는지
   - 마무리 턴에 "재회운 좋은 날?" → 방향 답 먼저 나오는지 (회피 첫 문장 금지)
   - 확답 질문 3연속(질문 바꿔가며) → 회피구 1회만 등장하는지
2. **추천 블록**: dev 배포 → 실 대화 → [END] → 결과 화면 추천 카드 → 클릭 → cross-type 진입 e2e.
2-b. **인챗 버튼**: [RECO:] 마커 대화 → 인라인 카드 노출 → "대화 더 할래"(카드 유지 확인) → 재클릭 → "마무리하고 넘어가기" → 그레이스풀 종료 → 결과 스킵 → 새 상담 진입 → 내 고민톡에서 기존 대화 결과·이어가기 확인.
3. **효과 측정**: `prompt_version` 갱신 (`2026-07-13-conversion-moment` 류) → 다음 분석에서 도달→결제 전환율 전후 비교 (baseline: 7.7%, 2026-07-13 findings). `previous_reading_id`로 추천 클릭→결제 어트리뷰션.

- **페르소나가 장사꾼처럼 느껴질 위험**: 크로스셀 대화당 1회 + 방향 답 뒤에만 + 선택권 문구 필수로 완화. QA에서 톤 확인.
