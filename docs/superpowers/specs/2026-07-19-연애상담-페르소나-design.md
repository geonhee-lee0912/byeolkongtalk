# 연애 상담 페르소나 v2 — 코어 판-중립화 + 릴레이션 하위 포함 (설계)

**작성일:** 2026-07-19 (W1 사이클 3c)

## Goal

연애 상담("우리 사이") 스레드 페르소나를 정식화한다. 공통 코어를 **판-중립 base**(톤·세이프티·화법)로 정리하고, "판을 보며 이야기하는" 규칙은 사주/타로 하위 문서로 귀속시키며, 연애 상담을 **코어를 상속받는 하위 문서**로 재작성한다. 스모크에서 드러난 3대 결함(판-정체성 미스핏, 심문 피로, off-topic 딜렉트)을 해소한다.

## 배경 / 진단

- 관계 페르소나 = `getCore()` + `byeolkong_relationship.md`. **v3 화법은 이미 상속됨** — 화법을 새로 짜는 게 아님.
- 스모크(`relationship.thread.happy_path`, 3c 착수 전 실행): 배관은 정상(등록→패스→chat→단언 통과, 별 60 정확 차감). 그러나 judge가 두 가지를 지적:
  1. **회피구 1회** — 커리어 concern에 "커리어 쪽은 별콩이가 짚는 영역 밖이라…"로 딜렉트(코어 §8 회피구 금지 위반).
  2. **심문 피로** — 턴1~5 대부분이 질문으로 종결(질문 2연속).
- 근인 3가지:
  1. **구조적 미스핏** — 코어 정체성이 *"사주판/타로를 펼쳐 읽어주는 존재"* 인데 **관계 스레드엔 판/카드 데이터가 없음.** 코어 §3 관찰형 훅의 "판 근거(일간·오행·카드)를 붙여" 규칙이 붕 뜸.
  2. **배관 갭** — saju/tarot는 W4에서 `computeTurnSignals`(직전 질문·단답 연속 → 동적 경고 주입)로 심문을 실전 강제하는데, **관계 chat 라우트는 이걸 안 씀** → 정적 심문 규칙이 실전에서 안 새게 못 막음. (심문 피로의 직접 원인.)
  3. **off-topic 규칙 부재** — 연애 인접 주제(일·가족 스트레스 등)를 관계 렌즈로 받는 규칙이 없어 딜렉트로 흘러 회피구가 됨.

## 아키텍처 결정

**채택: 코어 판-중립화 + 릴레이션 하위 포함.**
- 코어 = 톤·세이프티·화법의 단일 base (판-중립).
- "판을 보며 이야기하는" 정체성/근거는 사주·타로 하위 문서로 귀속.
- 릴레이션은 코어를 상속받는 하위 문서 — **판을 안 얹으니 프롬프트에 판 텍스트가 없음 = 누출 0.**
- 위기 안전망·금지 표현·화법 v3는 코어 단일 원천으로 전 페르소나 공유 → 안전 규칙 드리프트 없음, 캐릭터 일관성.

**기각안:**
- 순수 오버라이드(코어 불변, 도메인에서만 덮기) — 코어가 "판 읽어라"를 계속 말해 §3 훅에서 판-근거 누출 잔여.
- 세이프티만 별도 파일 추출 — 어정쩡. 코어에 판 정체성이 남아 릴레이션이 여전히 판 상속.
- 완전 스탠드얼론 릴레이션(전부 복사) — 위기 안전망·금지표현이 중복돼 hotline 번호 변경(이미 2024 109 통합) 시 사본 스테일 = 위기 유저가 틀린 번호 받는 **안전 사고 리스크**.

**회귀 리스크 검증(도메인 파일 정독 결과):**
- `byeolkong_saju.md`: "사주 상담 중… 사주판(팔자)을 근거로" + 훅 근거 "일간·오행"을 **자체 명시**.
- `byeolkong_tarot.md`: "타로 상담 중… 뽑은 카드를 근거로" + 훅 근거 "카드"를 **자체 명시**.
- `byeolkong_verdict.md`: **판 없는 대화형**(유저가 전한 양쪽 입장으로 판정) — 코어 중립화가 오히려 정합. 안전망은 코어에서 계속 상속.
- `byeolkong_fortune.md`: "공통 코어 위에 얹는" 표기 없는 **자체완결 페르소나**로 보임 → 코어 변경 무영향(구현 시 빌더가 `getCore()` 미사용인지 확인).
- 결론: 코어의 판 정체성은 **보강일 뿐 유일 원천이 아님** → 판-중립화의 회귀 리스크가 전 도메인에서 낮음.

## 설계

### 1. `byeolkong_core.md` — 판-중립 base로 정리 (삭제 아님, 판-특정 표현만 일반화)

- **§정체성** (사용자 확정 문구): 판-리더 배타 표현 제거 → *"사용자의 고민이나 일상을 듣고 친절하고 즐겁게 상담해주는 존재야, 그 과정에서 필요하면 사용자의 사주판이나 사용자가 뽑은 타로 카드를 펼쳐 흐름과 가능성을 읽어줄 수 있어."* (상담사 우선, 판은 필요 시 도구 — saju/tarot에도 안전.)
- **§화법 §3 관찰형 훅 근거**: "항상 판의 근거(일간·오행·카드 이름)를 붙여" → *"항상 근거를 붙여 — 사주면 일간·오행, 타로면 카드, 판이 없는 대화형(연애 상담)이면 유저가 한 말·관계 기억."*
- **§4 티키타카 이모티콘** (사용자 확정): "한 응답에 0~1개" → **"과하지 않게 적당히 사용 가능"** 으로 완화.
- **§5 턴 마무리 → 4택** (사용자 확정): 기존 3택(구체 질문 / 다음 볼거리 예고 / 소신 정리)에 **④ "친구와 대화하듯 자연스럽게 공감하며 대화 이어가기"** 추가 — 연애 상담 톤에 특히 정합, verdict-forward 완화.
- **§"언제·될까"** 등 "판이 기운 방향" 류 표현 → "네가 본 방향"으로 도메인-중립화.
- **유지(불변)**: 화법 v3(답먼저·완곡·티키타카·마무리·회피구 금지·심문 방지·디테일 재사용·호칭) + **위기 안전망 §** + **금지 표현 §** + 후속 대화 리듬. 코어 = 톤·세이프티·화법.

### 2. 판-도메인 자체 정체성 보강 (최소)

- `byeolkong_saju.md` / `byeolkong_tarot.md`: 이미 자체 명시 → 정체성 문장이 코어 의존이면 자체 문장으로 한 줄 보강("너는 사주판을 펼쳐 읽는 별콩이야" / "카드를 펼쳐 읽는 별콩이야"). **판 읽기 규칙은 불변.**
- `byeolkong_fortune.md`: 무변경(자체완결). 구현 시 빌더 확인.
- `byeolkong_verdict.md`: 무변경. 안전망·회피구 금지는 코어에서 계속 상속.

### 3. `byeolkong_relationship.md` — 재작성 (코어 상속 하위 문서, 판 미얹음)

- **정체성**: 이 관계를 계속 지켜보며 같이 고민하는 친구. 첫 문장 (사용자 확정): *"친구의 연애 내용에 대한 일상적인 대화 겸 상담 중이야."* 근거 = 대화 + 관계 파일/기억 (판 아님).
- **관찰형 훅 근거 = 대화·기억** 명시(코어 §3 대화형 가지). 은은한 '결/흐름' 은유는 캐릭터 톤으로 유지(voice A), **없는 사주판/카드 읽는 척 금지**(v1 규칙 유지·강화).
- **off-topic 흡수 규칙 신설**: 연애 인접 주제(일·가족 스트레스)는 딜렉트 말고 관계 렌즈로 받음(*"그 일 스트레스가 둘 사이에도 번지는 것 같아?"*). 완전 무관(코드 짜줘 등)만 부드럽게 관계로. **회피구("영역 밖") 금지**.
- **v1 배관 유지**: 관계 파일 데이터 규칙, 첫 진입/복귀 안부, 처방→체크인 루프, 스킬 제안(`[SKILL:key]`), `[END]` 없음, 소프트캡 마무리 톤.

### 4. `lib/claude.ts`

- `buildRelationshipSystemMessage`에 `turnSignals` 파라미터 추가 → `dynamicPart`에 `buildTurnSignalBlock(turnSignals)` 주입(saju/tarot 패턴 준용).
- 조립(`getCore()` + 도메인)은 **불변** — 코어 내용만 중립화되므로 어셈블리 코드 변경 없음.

### 5. 관계 chat 라우트 (`/api/relationship/chat`)

- `computeTurnSignals(과거 메시지, 이번 유저 발화)` 계산 → `buildRelationshipSystemMessage`에 전달. (심문 피로 근본 수정.)

### 6. 코어–연애상담 결 조정 (공유 코어의 점술-forward 부분)

공유 코어를 택했으므로, 점술 상담에 최적화된 코어의 일부가 캐주얼 연애 상담과 결이 어긋나는 지점을 **릴레이션 도메인 파일에서 재조정**한다(도메인 레이어의 역할). 사용자의 코어 편집(친근한 정체성 · 마무리 4택 · 이모티콘 완화)이 코어 자체도 이미 눅여 saju/tarot에도 안전.

- **§3 관찰형 적중 훅(첫 풀이 = 판 근거로 유저를 읽으며 시작)** → 연애 상담은 첫 턴이 훅이 아니라 **안부**로 열림(이미 아는 사이). 릴레이션 §첫 진입/복귀 안부가 오버라이드. 훅은 신규 등록 첫 진입에서만 관계 파일 기반 가벼운 관찰로 선택적.
- **§"언제·될까"(답 먼저·소신 예측)** → 방향은 주되(회피 금지) 톤은 verdict기계 아닌 공감-먼저(사용자 §5 4택 + 캐주얼 프레이밍). "흐름·가능성"으로 부드럽게(코어 §완곡).
- **용어 "풀이·판·적중"** → 연애 상담 출력엔 대화 용어. 릴레이션 도메인이 "이건 풀이가 아니라 대화"임을 명시.

원칙: **친근함 방향으로 눅여도 되는 항목은 코어에서**(saju/tarot도 안전), **saju/tarot에 꼭 필요한 점술-forward는 릴레이션 도메인 오버라이드**로. 3d 회귀로 양쪽 검증.

## Out of Scope

- verdict 화법 재설계(별도 스킬, 짧은 5턴 수렴형) — 코어 중립화로 인한 상속 정합만.
- 연애 상담 독립 화법 튜닝(공유 코어 채택으로 포기 — 대신 단일원천·일관성).
- 사주/타로/운세 화법 변경(정체성 한 줄 보강 외).

## 검증 (3d에서 실행)

- **관계 시나리오**(3a 하네스 `relationship.thread.*` + `relationship.verdict.*`): ①회피구 0회 ②심문 피로 없음(질문 2연속 없음) ③off-topic 관계 렌즈 수용 ④연속성(기억 인용) 체감 ⑤위기 안전망 정상.
- **판-도메인 회귀(필수)**: saju/tarot/fortune/verdict가 코어 중립화 후에도 판 읽기·안전 규칙 정상(judge 통과).

**성공 기준:** 위 검증 통과 + judge 핵심 차원(소신·완곡·회피구·심문·위기) 통과.

## 리스크 & 완화

- 코어 중립화가 판-도메인 판-정체성을 약화 → 도메인 자체 명시로 보강 + 3d 전수 회귀로 확인.
- turnSignals 주입이 관계 톤을 과교정(질문 과소·건조) → 3d에서 관찰, 필요시 관계용 신호 임계 조정.
