# 연애 상담 탭 리네임 + 리디자인 — 기술 설계 (2026-07-18)

> "우리 사이" 탭 이름이 직관적이지 않고, 활성 화면 상단이 난잡하며 대화창이 작다는 피드백에서 출발.
> 범위: **UI 표기 / 레이아웃 / 패스 표시 단위** 변경. DB 스키마·라우트·RPC·페르소나·안전망은 불변.

## 1. 목표

1. **탭 명명 체계 정리** — "-톡"의 의미 혼선 제거 + "우리 사이" 직관화
2. **활성 화면(S3/S4) 상단 1행화 + 대화 영역 최대**
3. **패스 = 시간권** 표시 모델 (남은 시간 시/분 표기)
4. **미등록 소개 화면(S1) 카피 정리** (구조는 유지)

## 2. 탭 명명 체계

**원칙**: "-톡" = 별콩이와의 **대화형 상담**. 리포트류엔 "-톡"을 안 붙인다. (단 연애 탭은 기능 직관 우선으로 '상담' 채택 — '연애톡'보다 '연애 상담'이 뭘 하는지 더 분명하다는 사용자 결정.)

| 현재 | 변경 | 정체 |
|---|---|---|
| 고민톡 `/` | **고민톡 (유지)** | 타로 기반 대화 상담 — 대화형이라 '톡' 유지 |
| 사주톡 `/fortune` | **별콩 운세** | 운세 리포트 진열(원샷) — '톡' 아님 → 제거 + '별콩 상점'과 짝 맞는 브랜드형 |
| 우리 사이 `/relationship` | **연애 상담** | 연애 지속 대화 — 기능 직관 우선 |

- 최종 탭바: **고민톡 · 별콩 운세 · 연애 상담 · 별콩 상점 · 내 정보**

**변경 파일 (리네임)**

## 3. 활성 화면(S3/S4) 레이아웃 — A안

현재 고정 상단 4겹(헤더카드 → SkillChipRow → partner 배너 → 상태줄)이 `ThreadChat`(`flex-1`)을 눌러 대화창이 작다. → **상단 1행**으로 축소.

### 3.1 헤더 (1행)
```
[연애 상담            ]              [ 48/72시간 | 패스권 구매 › ]
[자기 ❤ 연애 중  ✏️ ]
```
- **좌 클러스터**: 타이틀 `연애 상담` (bold) + 아래 `{호칭} ❤ {상태라벨}` + **✏️** (호칭·상태 바로 옆). ✏️ 탭 → 기존 `RelationshipEditModal`.
- **우 영역**: `{남은시간}/{총시간}시간 | 패스권 구매` (§4). 탭 → 패스 시트(기존 `PassSheet`, `setShowPassSheet(true)`).
- 좁은 폭 대응: 호칭이 길면 좌 클러스터 truncate. 우 영역은 `whitespace-nowrap` 우선.
- **삭제**: 기존 `page.tsx`의 별도 상태줄(`오늘 N번 남음 · 패스 D일 남음 · 패스 연장·구매`) — 정보는 우 영역(시간) + 패스 시트(오늘 대화 수)로 이관.

### 3.2 스킬 진입
- 상시 노출 `SkillChipRow`(헤더 아래 칩 행) **제거**.
- **⚡ 번개 아이콘**을 `ThreadChat` 입력창 **왼쪽**에 배치 → 탭 시 **스킬 시트** 오픈 (관계 체크인 ⭐45 · 걔 속마음 ⭐40 · 우리 궁합 ⭐40 · 싸움 잘잘못 판정 ⭐30).
  - 시트 콘텐츠·디스패치는 기존 `RELATIONSHIP_SKILLS`(`lib/relationship/skills.ts`) + `useSkillLaunch` 재사용. `SkillChipRow`의 접힌 메뉴를 시트 형태로 재활용.
  - ⚡ 아이콘 = MDI flash `M11,15H6L13,1V9H18L11,23V15Z`.
- **보조 발견 경로(유지)**: 페르소나가 대화 중 `[SKILL:key]` 마커로 제안 칩을 흘림(기존 파서/칩). 상시 노출을 없애도 발견은 이 둘로 커버.

### 3.3 partner 누락 배너
- 로직 유지(`partnerProfileId === null`). 위치만 헤더 바로 아래 **얇은 1줄**. 탭 → 수정 모달.

### 3.4 대화 영역
- 기존 `ThreadChat className="flex-1 min-h-0"` 유지. 상단 축소분만큼 자연 확대(추가 작업 없음).

## 4. 패스 = 시간권 표시 모델

**핵심**: 실제 기간(`PASS_PLANS.days` = 1/3/7)과 RPC·DB는 불변. **표시 단위만 시간**으로.

### 4.1 상품 라벨 (병기)
`lib/relationship/types.ts` `PASS_PLANS[].label` 변경:
| kind | days | cost | 새 label |
|---|---|---|---|
| day1 | 1 | 20 | **24시간(1일)** |
| day3 | 3 | 40 | **72시간(3일)** |
| day7 | 7 | 60 | **168시간(7일)** |
- 168시간이 안 익숙하므로 **"(N일)" 병기** — 구매/이용권 표시 지면에 적용.
- `kind`/`days`/`cost` 불변 → `purchase_relationship_pass` RPC·`PASS_PLAN_BY_KIND` 그대로.

### 4.2 남은 시간 표기 (헤더 우 영역)
`expiresAt` 기준 새 util `formatPassRemaining(expiresAt, planDays)` (기존 `formatPassDDay` 대체, `page.tsx`):
- 총시간 = `planDays * 24`.
- **1시간 이상**: `{남은시}/{총시}시간` (예 `48/72시간`). **시간 단위로만** 갱신(48→47→…).
- **마지막 1시간(남은 < 60분)**: **10분 단위** 분 표기 `{50·40·30·20·10}분/{총}시간` (예 `40분/72시간`). 남은 < 10분 → `10분` 유지(0 직전까지) 후 만료.
- **재구매 스택**(남은 > 총시간)일 때: 분수 없이 `{남은}시간 남음` (예 `82시간 남음`).
- 갱신 방식: 페이지 로드시 계산 + 필요 시 분/시 경계 타이머(과한 실시간 틱 불필요 — "시간 단위로만").

### 4.3 패스 시트 (`components/relationship/PassSheet.tsx`)
헤더 `패스권 구매`(및 남은시간 영역) 탭 → 시트:
- `지금 패스` — `72시간권 · 48시간 남음`
- `오늘 남은 대화` — `{allowance-used} / 대략 20번` (기존 `daily` 데이터)
- **오늘 연장** — `💬 오늘 5번 더 (제한 없이) · ⭐5` → 기존 `rel_extend` (EXTEND_COST=5, EXTEND_TURNS=5)
- **구매·연장** — 24/72/168시간권 (병기 라벨, ⭐20/40/60). 활성 중 구매=이어붙임(기존 RPC).
- 확인은 기존 `PassConfirmModal` 경유.

### 4.4 미활성 상태(S2)
- `PassPanel` 그대로 메인 CTA(구매 크게 노출). 시간권 라벨만 반영.

## 5. 하루 대화 한계 → 칩

- 소프트캡(`dailyTurnAllowance`, 기본 20턴) 도달 시 `ThreadChat`의 capReached UI를 **칩**으로 교체:

## 10. 리스크 / 열린 점

- **헤더 폭**: 좌 클러스터 + `48/72시간 | 패스권 구매`가 좁은 기기에서 빠듯 → 호칭 truncate + 폰트/여백 튜닝 필요(구현 시 브라우저 확인).

- (선택) `/fortune` 히어로 "사주 운세" 표기 정렬 여부는 후속 판단.
