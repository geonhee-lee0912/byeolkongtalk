# 별콩 운세(/fortune) 홈 스타일 카드·내용 구성 통일 — 설계

> 작성일 2026-06-22 · 범위: `app/fortune/page.tsx` + `lib/fortune/types.ts` · 홈(고민 상담, `/`)의 카드·내용 구성을 운세 페이지에 적용

## 1. 배경 / 목적

하단탭 **고민 상담 페이지(`/`, 홈)** 의 고민 카드는 그라데이션 타일 + 제목 + 설명 + 해시태그 칩에 "이렇게 사용해요" 안내 박스를 갖춘 풍부한 구성이다. 하단탭 **별콩 운세(`/fortune`)** 의 운세 목록 카드는 (직전 작업으로 흰 카드+그림자·흰 활성탭까지는 통일됐지만) 여전히 단색 lilac 타일 + 이모지 + 한 줄 tagline으로 단조롭다. **디자인뿐 아니라 내용 구성도 홈과 유사하게** 맞춘다.

## 2. 결정 사항 (확정)

| 항목 | 결정 |
|---|---|
| 그룹핑 | **사주/타로 탭 유지** (의미 있는 구분). 인기/다른 섹션화는 하지 않음 |
| 카드 표면 | 흰 카드 + 그림자 + 흰 활성탭 — **직전 작업(`c0a9078`)으로 이미 적용됨**, 유지 |
| 타일 | 단색 `lilac-soft` → **운세 종류별 그라데이션 타일** (이모지/붉은말 아이콘은 유지) |
| 해시태그 | 운세 종류별 **#태그 칩 신규 추가** |
| 설명 | 기존 `tagline`을 홈처럼 제목 아래 설명으로 강조 배치 |
| 비용 배지 | 무료/⭐ 배지 **유지** (운세 전용) |
| 활성 카드 `›` | **제거** (홈 카드처럼 chevron 없음). 비활성 "준비 중"은 유지 |
| 안내 박스 | 탭 위에 **"💡 이렇게 사용해요" 그라데이션 안내 박스 신규** (홈과 동일 톤) |
| 이미지 아이콘 | 도입 안 함 (에셋 없음) — 이모지 유지 |

## 4. 화면 구성 (`app/fortune/page.tsx`)

```
히어로 (별콩이 + "별콩 운세" + 부제)        ← 변경 없음
FortuneGeneratingList                       ← 변경 없음
[💡 이렇게 사용해요] 안내 박스               ← 신규
사주 / 타로 탭 (흰색 활성)                   ← 이미 적용, 유지
운세 카드 리스트 (홈 스타일)                 ← 카드 리뉴얼
하단 주의문구 ("운세는 정해진 미래가…")      ← 변경 없음
```

### 안내 박스 (신규)
홈의 "이렇게 사용해요" 박스와 동일 톤:
- `linear-gradient(135deg, #F6EFFF 0%, #EFE6FB 50%, #FBEFF4 100%)` 배경 + `border border-lilac/40` + rounded-2xl
- `💡 이렇게 사용해요`(lilac-deep, 작게) + 안내 문구: "별콩 운세는 대화가 아니라, 생일·고민을 입력하면 별콩이가 한 장의 리포트로 정리해주는 방식이에요. 가볍게 골라봐요."
- 위치: 탭 위 (`FortuneGeneratingList`와 탭 사이)

### 운세 카드 (리뉴얼)
기존 카드 레이아웃(흰 카드 + 타일 + 라벨/배지 + tagline)을 홈 카드 구조로:
- **타일**: `w-12 h-12 rounded-xl` 유지, 배경 `bg-lilac-soft/60` → `style={{ background: FORTUNE_GRADIENTS[f.type] }}`. 안에 이모지/`RedHorseIcon` 그대로.
- **제목 + 비용 배지**: 기존 행 유지 (label + 무료/⭐ 배지).
- **설명**: `tagline`을 제목 아래 `text-[12.5px] text-text-light/80 leading-snug` 으로 (홈 설명 톤).
- **해시태그 칩**: 설명 아래 `flex flex-wrap gap-1 mt-2`, 각 칩 `text-[11px] font-bold text-lilac-deep bg-lilac-soft/60 px-2 py-0.5 rounded-full`, `#{tag}` (홈과 동일 스타일). `FORTUNE_HASHTAGS[f.type]`.
- **`›` chevron**: 활성 카드에서 제거. 비활성(`!f.active`)은 "준비 중" 텍스트 유지.
- 카드 컨테이너 흰색+그림자(`shadow-[0_2px_10px_rgba(159,138,208,0.08)]`)·활성/비활성 분기는 직전 작업 그대로 유지.

## 5. 구현 범위 / 비범위

- **범위**: `lib/fortune/types.ts`에 두 맵 추가, `app/fortune/page.tsx`에 안내 박스 + 카드 리뉴얼(그라데이션 타일·설명·해시태그·chevron 제거).
- **비범위**: 사주/타로 탭 동작, 히어로, FortuneGeneratingList, 하단 주의문구, 무료/유료 로직, 운세 상세/입력 화면, 다른 페이지. 이미지 아이콘 에셋 도입.

## 6. 성공 기준

- 운세 카드가 홈 고민 카드와 같은 구성(그라데이션 타일 + 제목 + 설명 + 해시태그)으로 보인다.
- 무료/⭐ 비용 배지가 유지되어 과금 정보가 그대로 노출된다.
- 사주/타로 탭으로 도메인 구분이 유지된다.
- "이렇게 사용해요" 안내가 홈과 같은 톤으로 노출된다.
- 빌드 통과 + dev 브라우저에서 사주/타로 탭 모두 카드·해시태그·안내 박스 육안 확인.
