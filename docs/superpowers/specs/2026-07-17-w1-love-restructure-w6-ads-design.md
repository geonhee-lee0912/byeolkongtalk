## 0. 한 줄 요약

전환의 75%가 "관계 스프레드 갭 결제"에서 우연히 나오는 현실을 의도된 설계로 전환한다 — **고민톡=타로 전용 대화 상담(연애 존 전면), 사주=운세 리포트 진열대**로 방식 패밀리를 분리하고, 지속 대화형 신설 종목 "우리 사이"(패스+스킬)로 리텐션·재결제 엔진을 만들고, 광고 소재·랜딩을 이 구조와 한 세트로 정렬한다.

## 1. 구조 결정 — (d) 방식 패밀리 분리

유저가 방법(사주/타로)을 고르는 현행 구조를 폐기하고 두 패밀리로 분리:

| 패밀리 | 방식 | 성격 | 탭 |
|---|---|---|---|
| **고민톡** | 타로 전용 | 대화형 상담 (태그 → 고민 입력 → 스프레드 선택 → 채팅) | 고민톡 |
| **운세** | 사주 전용 | one-shot 리포트 (입력 → 결과지) | 사주 |

**근거 (prod 쿼리, 제외 6명 반영)**:
- 대화 상담에서 사주는 리딩 15%(27/181)·주별 비중 하락(19%→12%)·현금 결제자 기여 1/12명
- 운세 리포트에선 사주가 우세 (유료 별: 사주 330 vs 타로 220, 유료 1위 = 궁합 9건 270별)
- 사주 상담 상품별 완주: 성향 10/14, 좋은날 5/9, **오늘글자 0/3**, 선택비교 1건 — 대화 체질이 아님
- 타로 채팅에 생년월일 2인분을 붙여넣은 궁합 수요 미스매치가 구조적으로 해소됨

**교차 연결**: 인챗 RECO(수락 0/23, 죽은 통로)가 아니라 **정적 카드**로 — 연애 존에 궁합 카드, 타이밍 태그에 좋은 날 링크, 인간관계 태그에 관계궁합 링크.

**스프레드 선택(깊이=가격) 유지**가 이 안의 조건: 유저 선택권과 가격 유연성 보존 (질문 중심 (a)안 기각 사유였음).

## 6. 신설 종목 — "우리 사이"

**정체성**: 지속 대화형 연애 상담 에이전트 + 호출형 스킬. 세션 개념 없음 — 상대 등록 시 **영원한 스레드 1개** 생성, 한탄·해결·장기 기억 대화가 이어짐.

- **상대 등록** (최초): 호칭 + 관계 상태(썸/연애중/이별/짝사랑) + **양쪽 생년월일(시) 등록 유도** (내 것 포함, 궁합 스킬 조건). 관계 파일 1차엔 1개.
- **v1 스킬 4종** (마커+UI 칩, 기존 자산 재사용): 싸움 잘잘못 판정 30별(신규, 대화형 — 양쪽 입장 청취→비율 판정→화해 처방) / 관계 체크인 45별(신설 스프레드 재사용) / 우리 궁합 40별(등록 생년월일로 호출) / 걔 속마음 40별(신설 스프레드 재사용). 판정 30별은 §7 원칙의 의도적 예외 — 스킬은 패스 선구매(웰컴 소진) 후의 인챗 상품이라 웰컴 함정이 성립하지 않음.
- **과금 = 패스** (별 소모형 기간권, `expires_at`): **1일 20별 / 3일 40별 / 7일 60별**. 40=웰컴30+star_10 정확, 60=웰컴30+star_30 정확. 일당 단가 체감(1,860/1,240/797). 등록 증정 없음(트라이얼=웰컴별로 1일권). 7일권에 "추천" 뱃지. 30일권은 갱신 데이터 확인 후.
- **한도**: 일일 소프트캡 20턴 + 초과 시 별콩이 부드러운 마무리 + **5별 당일 연장**. ⚠️ **한도는 구매 화면에 명시** ("하루 최대 20번의 대화, 초과 시 별로 연장") — 무제한 오인/환불 분쟁 방지.
- **원가 안전성** (Sonnet 5 $3/$15, 턴당 ~₩25): 7일 최악(140턴) ₩3,500 vs 매출 ~₩5,580.
- **처방→체크인 루프**: 대화 중 자연 처방 → **인앱 체크인** (다음 방문 시 별콩이가 먼저 안부). 카카오 푸시는 W5.
- **최대 공사 항목**: 장기 기억 인프라 (수백 턴 스레드의 롤링 요약 → 관계 파일 적립). 기존 reading 단위 구조에 스레드 개념 추가 필요.
- 스킬 제안 멘트에 탈출구 문구 금지 (C3 교훈).

기존 /start 패턴 재사용. 구성: 히어로(소재와 동일 카피) → **"네가 보고 온 그 고민" 하이라이트 카드 + 메인 CTA** → "다른 연애 고민이라면" 나머지 연애 태그 5종 목록(어느 것이든 로그인→해당 태그 고민 입력) → 하단 "연애 말고 다른 고민 →" 홈 링크. 고민 입력 화면에서도 태그 변경 가능. 랜딩 = 연애 존 미니 홈.

### 소재 믹스 수정 방향 (2026-07-18, 사이클 2 후속 논의 — ⚠️미확정·사이클 3 착수 시 재브레인스토밍 대상)

> 위 "소재 3본(tarot+연락+재회)"을 아래로 바꾸는 방향이 논의됨(사용자 지시). **아직 어느 계획에도 미포함.** 사이클 3 착수 시 **소재 문구·이미지·영상 형태까지** 세세 브레인스토밍 후 확정한다. 개수는 3본 유지(Meta 학습 볼륨 제약 존중, 구성만 교체).

| 슬롯 | 소재 | 훅 | 랜딩 | 목적·측정 |
|---|---|---|---|---|
| 1 대조군 | tarot 무수정 (유지) | 현행 | 홈 | 벤치마크·학습 보존 |
| 2 범용 연애 (신규) | 특정 태그(재회/연락) 대신 **캐치올** | "그 사람의 마음이 궁금해?" (= '걔 속마음' 의도적 모호 태그) | 연애 존(홈 `/` 또는 신규 `/start?v=love`) | 획득 볼륨(넓은 그물·싼 클릭·학습 볼륨) |

- ⚠️ **리스크**: 슬롯2 범용 연애가 tarot 대조군과 훅이 겹치면 인크리멘탈 학습 소실 + 자기 옥션 경쟁(CPM↑) → 둘 명확 분리 필요(대조군=제네릭 유지). **미해결 확인 항목**: 현행 tarot 대조군 소재가 이미 연애 broad 훅인지 확인 후 슬롯2 각도 확정.
- 원래 계획의 "연락 vs 재회 A/B"는 폐기 → 대신 "제네릭(대조군) vs 범용연애 vs 깊은상품(연애상담)" 3자 비교.
- 사이클 3 코드 팔로업(소량): ① 범용 연애 랜딩(홈이면 무코드 / `v=love` variant면 소량) ② `/relationship` 콜드스타트에 광고 **어트리뷰션 파라미터**(utm/v) 수신 신설 ③ `utm_content` 개별화. 소재 제작·지면(IG 피드+릴스+스토리)·counsel OFF·`ad_spend.clicks` 입력은 사용자 액션.
