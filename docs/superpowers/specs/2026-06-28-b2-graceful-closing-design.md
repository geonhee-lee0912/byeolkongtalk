# B-2 마무리 완충 (Graceful Closing) 설계

> 작성일: 2026-06-28
> 상태: 설계 승인 → 구현
> 출처: QA 파일럿(90 케이스)에서 심판 위반 최다(마무리 적절성 10건)

## 문제

abs cap(사주 9턴 / 타로 9~13턴) 도달 시 별콩이가 사용자 상태와 무관하게 *"언제든 다시 와"* + `[END]`로 닫는다. 사용자가 그 시점에 고민을 열어둔 상태(예: "무섭다", "뭘 하고 싶은지 모르겠다")면 **당혹스러운 강제 종료**가 되어 리텐션을 해친다.

핵심 통찰: 어색함의 원인은 **턴이 모자라서가 아니라 닫는 방식(화법)**이다. 따라서 주 레버는 마무리 화법이고, 연장은 보조다.

## 설계 (① 적응형 연장 + ② 그레이스풀 브리지)

### 감지 = 모델 판단 (코드 휴리스틱 없음)
수렴 로직이 이미 전부 페르소나 프롬프트 기반(`lib/claude.ts` wrap 가이드)이고, 파일럿에서 별콩이는 미해결을 이미 인지했다("더 깊은 곳에서 힘든지"). 행동 지시만 없었음 → 새 감지 인프라 불필요. 마지막 user 턴의 개방/미해결 신호를 모델이 읽고 판단한다.

### ① 적응형 연장 (+2턴, 미해결 감지 시에만)
- **임계치 상향** — absTurnCap에 연장 예산 +2:
  - `lib/saju/constants.ts`: `ABS_TURN_CAP` 9 → 11
  - `lib/tarot/constants.ts`: one_card 9→11, two_card 10→12, three_card 11→13, relationship_5 13→15
- 자연 hardcap(turn+chars)은 그대로 두되 **의미를 바꿈**: "강제 종료" → "마무리 시작(상황 보고)".

### 자연 hardcap vs 절대 cap 분리 (코드 로직)
현재 `buildSystemMessage`/`buildTarotSystemMessage`는 자연 hardcap과 absCap을 모두 `mode="hardcap"`(강제 [END])으로 합쳐버린다. 이를 분리:
- **naturalHardcap** (turn+chars 도달, but < absCap): 그레이스풀 마무리 시작. **단, 사용자가 방금 진짜 미해결 고민을 열었으면 그 매듭을 이번 턴에 짚고 [END] 보류 가능** (absCap까지 한두 턴 더). 정리되는 분위기면 그레이스풀 브리지로 [END].
- **absHardcap** (>= absCap): 반드시 이번 턴 종료. 그레이스풀 브리지 + [END].

### ② 그레이스풀 브리지 마무리 (핵심 — wrap 가이드 재작성)
absCap/마무리 턴 응답이 지켜야 할 화법:
- 미해결 매듭을 **콕 짚어 인정** ("그 '무섭다'는 마음, 지금 다 풀긴 어려운 주제인 거 알아")
- "끝났으니 또 와"(내치는 톤) **금지** → **"이건 다음에 새로 펼쳐서 같이 더 봐도 좋아"**(이어짐 약속, 아래 범위 주의)
- 사용자가 *덜 풀린 채 끊겼다*가 아니라 *여기까지 같이 왔고 다음이 있다*고 느끼게
- 기존 금지 표현 유지: "감사/마지막 질문/오늘은 여기까지" 등 절벽감 문구 X

### 마무리 약속의 범위 (빈 약속 회피)
현재 제품은 [END] 이후 대화 이어가기 기능이 **없다**(타로 미완료 reading resume만 존재). 따라서 브리지 문구는 **"새로 펼쳐서 또 보자"**(이미 지원되는 새 reading + 히스토리)로 프레이밍한다. 리터럴 "이어가기"는 별도 기능(백로그)으로 분리하며, 그 기능이 나오면 문구를 "이어가기 눌러서 이어보자"로 업그레이드한다.

## 범위
- B-2 = wrap 가이드 문구 + 임계치(3파일)만. 코드 로직은 naturalHardcap/absHardcap 분리.
- **범위 밖(백로그 기록 완료)**: "고민 이어가기" 연속성 실기능, 사주 resume 파리티.

## 변경 파일
- `lib/saju/constants.ts` — ABS_TURN_CAP 9→11
- `lib/tarot/constants.ts` — 스프레드별 absTurnCap +2
- `lib/claude.ts` — `buildSystemMessage`(사주) + `buildTarotSystemMessage`(타로) wrap 가이드: naturalHardcap/absHardcap 분리 + 그레이스풀 브리지 문구
