# 광고 전용 랜딩 `/start` 설계

날짜: 2026-07-04
상태: 사용자 승인됨 (브레인스토밍 세션)

## 배경 / 문제

인스타 광고 딥링크 점검 결과 (플랜 `2026-07-04-ad-launch.md` Task 9 의 딥링크 3개):

- 소재1 상담 → `/` : 정상이지만 광고 맥락(고민 상담 훅)이 홈 일반 화면에 희석됨
- 소재2 운세 → `/fortune/daily` : 생년월일 입력을 다 채운 뒤에야 로그인 요구 → 로그인 왕복 시 입력 유실, 이탈 유발
- 소재3 타로 → `/tarot` : `sessionStorage` PENDING 전제라 콜드 진입 시 즉시 홈으로 튕김 — **광고 랜딩으로 사용 불가**

→ 광고 유입 전용 랜딩 한 페이지를 만들어 세 소재를 모두 수용한다.

## 핵심 결정 (사용자 확정)

1. **얕은 랜딩 + 기존 흐름 재사용** — 랜딩은 "광고별 메뉴 + 로그인 + 웰컴 안내"까지만 하고, 실제 상담/운세는 기존 다운스트림(`/concern`, `/fortune/*`)으로 핸드오프
2. **선택 → 로그인 순서** — 광고가 약속한 서비스를 먼저 고르게 해 관여 유도, 로그인은 선택 직후
3. **소재별 메뉴** (한 페이지, `utm_content` 분기):
   - `counsel` → 감정 분류 10종
   - `daily` → 별콩 운세 **전체 10종** (오늘의 운세 단일 아님)
   - `tarot` → "타로로 고민 상담 / 타로 운세 보기" 갈래 선택 먼저
4. **광고 전용 접근** — 유효한 `utm_content` 없이 직접 URL 진입 시 홈으로 replace (오가닉 유저 혼란 방지)
5. **웰컴 별 알림 = 큰 팝업(모달)** — 토스트 아님
6. **히스토리 백 보장** — 핸드오프 후 브라우저 뒤로가기가 랜딩(쿼리 유지)으로 자연 복귀. 페이지 내 "뒤로" 버튼 변경은 범위 밖
7. **결과 화면 업셀은 모든 유저** — 광고 유입 플래그 구분 없음
8. **재방문 넛지** — 로그인 + 리딩 0건 유저가 홈 진입 시 상단 배너 (랜딩으로 보내지 않음)
9. 랜딩 리본 카피는 "지금 가입하면 웰컴 별 30개" 통일 — "무료" 약속은 리본에서 제거, 가격은 서비스 카드가 표기

부수 완료: 소재2 이미지 서브 카피 "오늘의 운세 무료로 보고 가" → **"운세 리포트 무료로 보고 가"** (Figma 수정 + `ad-assets/final/` 재export 완료)

## 라우트 / 접근 가드

- 새 페이지 `app/start/page.tsx` (Client Component)
- 광고 딥링크 3개를 전부 교체:
  ```
  https://byeolkongtalk.com/start?utm_source=meta&utm_medium=paid&utm_campaign=launch_2026q3&utm_content={counsel|daily|tarot}
  ```
- 가드: `utm_content` 가 `counsel | daily | tarot` 가 아니면 `router.replace("/")`
- `AppShell` 예외 라우트에 `/start` 추가 — Header/BottomTab 숨김 (광고 맥락 집중)

## 페이지 구성 (variant 공통 뼈대)

```
[다크 히어로]  광고 카피 매칭 헤드라인 + 별콩이 이미지
[골드 리본]    "지금 가입하면 웰컴 별 30개 ✨"
[서비스 카드 리스트]  ← 카드 자체가 CTA (별도 시작 버튼 없음)
```

variant 별 카드 리스트:

| utm_content | 히어로 카피 | 카드 |
|---|---|---|
| `counsel` | "요즘 마음 복잡하지? 별콩이가 들어줄게" | `EMOTION_OPTIONS` 10종 (홈과 동일 스타일) |
| `daily` | "오늘 하루, 어떤 흐름일까?" | `FORTUNE_LIST` 10종 전체. **오늘의 운세만 맨 위 + "광고에서 본 그거" 뱃지**, 유료는 ⭐가격 뱃지 |
| `tarot` | "카드는 네가 직접 뽑아" | 갈래 2택: 🔮 타로로 고민 상담 / 🃏 타로 운세 보기 → 같은 페이지 내 스텝 전환 (상담 → 감정 10종, 운세 → 타로 운세 5종) |

기존 상수(`lib/emotions.ts`, `lib/fortune/types.ts`) 재사용 — 새 데이터 정의 없음.

## 선택 → 로그인 → 복귀 흐름

1. 카드 탭 → 선택 내용을 `sessionStorage("byeolkong:start_pending")` 저장
2. 비로그인이면 `/login?next=/start?{원래 쿼리 전체}` → 카카오 → `/start` 복귀
3. 복귀 시 `?welcome=1` (kakao 콜백이 신규가입에 이미 부여) 감지 → **웰컴 팝업**:
   - 별콩이 이미지 + "웰컴 별 30개 도착 ✨" + 현재 잔액 + "바로 시작하기" 버튼
   - 닫으면 `start_pending` 의 선택으로 자동 진행
4. 진행 매핑 (모두 `router.push` — 브라우저 백 = `/start` 복귀):
   - 감정 선택 → `sessionStorage("byeolkong:emotion")` 세팅 → `/concern` (기존 흐름 그대로)
   - 운세 선택 → 해당 `FORTUNE_CONFIG.href`
   - 타로 갈래 → 페이지 내 스텝 전환 (URL 유지)
5. 기로그인 유저가 카드 탭 → 로그인 스킵, 팝업 없이 바로 진행

엣지: `start_pending` 은 탭 단위(sessionStorage) — 로그인 후 다른 탭/브라우저로 복귀하면 선택이 없을 수 있음 → 팝업 닫은 뒤 pending 없으면 그냥 랜딩에 머묾 (재선택). sessionStorage 접근은 기존 패턴대로 try/catch.

## 재방문 넛지 배너 (홈)

- 위치: `app/page.tsx` 상단, 기존 "이어하기 배너" 패턴 재사용 (`/api/readings` 이미 fetch 중)
- 조건: 로그인 상태 && 리딩 0건
- 문구: "⭐ 웰컴 별 30개가 기다리고 있어 — 아래에서 첫 고민을 골라봐. 운세 리포트는 하단 **별콩 운세** 탭에서!"
- 동작: 클릭 시 감정 그리드로 스크롤 (랜딩 `/start` 로 보내지 않음 — 광고 전용 가드 유지)
- 첫 리딩 생성 시 자동 소멸 (조건 미충족)

## 결과 화면 업셀 (모든 유저)

`/saju/result` · `/tarot/result` · `/fortune/result` 하단에 공용 컴포넌트 1개 (`components/upsell/ResultUpsell.tsx` 가칭):

1. **첫 충전 배너** — `/api/stars/first-charge-status` 자격 조회, 미충전 유저에게만 "첫 충전 +50% 보너스" → `/shop`
2. **크로스셀 카드 2장** ("이런 것도 있어") — 정적 규칙, 개인화 없음:
   - 상담 결과 → 운세 2개 추천 (오늘의 운세 + 이번달)
   - 운세 결과 → 상담 진입 1개 + 같은 base(사주/타로)의 다른 운세 1개 (방금 본 것 제외, `FORTUNE_LIST` 순서상 다음 항목)

## 트래킹

- 랜딩은 기존 Meta Pixel PageView 로 충분. 소재별 성과는 메타 광고별 분해로 판독
- 추가 커스텀 이벤트 계측 없음 (YAGNI)

## 범위 밖 (명시)

- 각 페이지 내 "뒤로" 버튼을 `/start` 로 바꾸는 것 (브라우저 백만 보장)
- `/fortune/daily` 의 "입력 후 로그인" 마찰 자체 수정 — 광고 유입은 랜딩에서 로그인이 선행되므로 광고 시나리오에선 소멸
- 광고 유입 유저 플래그/코호트 저장
- 홈 `?welcome=1` 팝업 (랜딩만)

## 검증

1. `npx tsc --noEmit` 에러 0
2. dev 배포 후 소재 3종 URL 콜드 진입(시크릿) 각각:
   - 메뉴 노출 → 카드 선택 → 카카오 로그인 → 웰컴 팝업(잔액 30) → 서비스 도달
   - 서비스 화면에서 브라우저 백 → `/start` 복귀 (variant 유지)
3. `utm_content` 없이 `/start` 직접 진입 → 홈으로 replace
4. 리딩 0건 계정으로 홈 진입 → 넛지 배너 노출, 리딩 1건 생성 후 소멸
5. 결과 화면 3종에서 첫충전 배너(자격자) + 크로스셀 카드 노출
