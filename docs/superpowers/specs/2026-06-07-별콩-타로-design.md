# 별콩 운세 타로 신상품 설계 (Design Spec)

**날짜:** 2026-06-07
**프로젝트:** byeolkong_talk
**목표:** 별콩 운세(fortune) 섹션의 타로 탭에 "입력→단일 구조화 JSON 리포트" 모델의 타로 상품 5종을 추가한다. 카드 뽑기 의식은 기존 채팅 타로의 덱 UI를 공유 컴포넌트로 추출해 재사용한다.

---

## 1. 배경 & 모델

별콩 운세는 **원샷 구조화 리포트** 모델이다(사주/궁합과 동일): 사용자가 입력하면 백그라운드에서 단일 JSON 리포트를 생성하고 `/fortune/result`에서 렌더한다. 채팅이 아니다.

타로에는 두 갈래가 이미 존재한다:
- **(a) 채팅 타로** — `/tarot` 플로우 (concern → spread select → `/tarot/draw` → `/tarot/reading` 스트리밍 → `/tarot/result`)
- **(b) 원샷 fortune 타로** — 본 스펙이 다루는 대상. `tarot_oneshot`(active:false)을 신규 5종이 대체한다.

본 상품은 (b)에 속하며, **카드 뽑기 연출만** (a)의 덱 UI를 공유한다.

### 확정된 결정 (7가지)

| 항목 | 결정 |
|---|---|
| 모델 | 원샷 구조화 리포트 |
| 입력 | 고정 테마, 질문 텍스트 없음 |
| 축 | 무료 미끼 + 도메인 유료 혼합 |
| 카드 연출 | 풀 의식(`/tarot/draw` 덱 뽑기) **공유 컴포넌트로 추출** 후 재사용 |
| 무료 상품 | 오늘의 타로 한 장 (1장, 평생 5회 무료 → 이후 5별) |
| 유료 라인업 | 연애운 / 금전운 / 직장·진로 / 인간관계 (각 3장) |
| 가격 | 전부 15별 (**임시** — 추후 전체 가격 재논의 예정) |

---

## 2. 상품 5종 & FORTUNE_CONFIG

`lib/fortune/types.ts`의 `FortuneType`에 타로 도메인 5종을 추가한다. 기존 `tarot_oneshot`은 신규 5종이 대체한다.

| type | label | emoji | cardCount | cost | 무료정책 | base |
|---|---|---|---|---|---|---|
| `tarot_daily` | 오늘의 타로 한 장 | 🃏 | 1 | 무료 5회 후 5별 | 사주 daily 미러링 | tarot |
| `tarot_love` | 연애운 타로 | 💗 | 3 | 15별 | — | tarot |
| `tarot_money` | 금전운 타로 | 💰 | 3 | 15별 | — | tarot |
| `tarot_career` | 직장·진로 타로 | 💼 | 3 | 15별 | — | tarot |
| `tarot_relation` | 인간관계 타로 | 🤝 | 3 | 15별 | — | tarot |

- **emotionTag sentinel:** `fortune:tarot_daily`, `fortune:tarot_love`, … (FORTUNE_SENTINEL_PREFIX 그대로)
- **base:** `"tarot"` → `fortune/page.tsx`의 타로 탭에 자동 노출 (FORTUNE_TABS 필터 `f.base===tab`)
- **MAX_TOKENS_BY_FORTUNE:** `tarot_daily` 1024, 도메인 4종 각 2048
- **href:** `/fortune/tarot/[type]` (섹션 4 라우팅)
- **cardCount:** 별도 필드를 두지 않고 섹션 3의 `TAROT_POSITIONS[type].length`를 단일 진실 원천으로 쓴다(라벨과 카드 수가 항상 일치하도록).

> 가격은 임시값 15별(도메인) / 5별(오늘의 타로 유료 전환분)이다. 추후 전체 가격 브레인스토밍에서 조정.

---

## 3. 포지션 라벨 (도메인별)

`lib/fortune/types.ts`에 fortune 전용 맵 `TAROT_POSITIONS: Record<FortuneType, string[]>`를 추가한다. (채팅 타로 전용 `lib/tarot/spreads.ts`의 `SPREAD_LABELS`는 건드리지 않는다.)

| 도메인 | 카드 1 | 카드 2 | 카드 3 |
|---|---|---|---|
| 연애운 (`tarot_love`) | 나의 마음 | 상대의 마음 | 관계의 흐름 |
| 금전운 (`tarot_money`) | 현재 흐름 | 장애물 | 기회·조언 |
| 직장·진로 (`tarot_career`) | 현재 위치 | 변화의 기류 | 나아갈 길 |
| 인간관계 (`tarot_relation`) | 나 | 상대·무리 | 관계의 방향 |
| 오늘의 타로 (`tarot_daily`) | 오늘의 메시지 | — | — |

라벨의 3중 용도:
1. 카드 뽑기 슬롯 라벨
2. 리포트의 카드별 섹션 제목
3. AI 프롬프트의 포지션 컨텍스트

## 4. 플로우 & 카드 뽑기 의식 재사용

### 4.1 공유 컴포넌트 추출

`/tarot/draw/page.tsx`는 채팅 타로 전용(concern+spread 읽고 `/tarot/reading`으로 라우팅)이라 그대로 재사용 불가. 덱 UI를 순수 컴포넌트로 추출한다.

```
components/tarot/CardDrawRitual.tsx  (신규 — 순수 UI, 라우팅/저장 책임 없음)
  props: {
    cardCount: number,
    slotLabels: string[],       // 섹션 3의 포지션 라벨
    accent: string,
    onComplete: (drawn: DrawnCard[]) => void,  // 뽑기+방향 완료 시 호출
  }
```
- 포함: HorizontalDeck / SlotRow·Slot / 카드 비행 애니메이션(FLIGHT_MS=440) / 방향 토글(정/역)
- 제외(호출자 책임): StarConfirmModal, sessionStorage, 라우팅
- `DrawnCard` 인터페이스(`lib/tarot/spreads.ts`)는 그대로 재사용: `{ position, label, card_id, direction }`

### 4.2 기존 채팅 타로 리팩터 (회귀 위험 최소)

`/tarot/draw/page.tsx`는 `CardDrawRitual`을 쓰도록 변경:
- 기존처럼 `TAROT_SPREAD_KEY`를 읽어 cardCount/slotLabels 산출
- `onComplete(drawn)` → `TAROT_DRAW_KEY` 기록 후 `/tarot/reading`으로 라우팅
- **외부 동작은 동일**해야 한다(회귀 테스트 대상).

### 4.3 운세 타로 플로우

```
/fortune (타로 탭) → 상품 카드 클릭
  → /fortune/tarot/[type] 입력 화면
     → 상품 소개 + ⭐비용 + <CardDrawRitual cardCount slotLabels accent onComplete />
     → 카드 다 뽑고 방향 확정 "타로 보기"
        → tarot_daily 무료 잔여 있으면: StarConfirmModal 생략, 바로 생성
        → 그 외: StarConfirmModal (auth 확인 + 잔액 조회)
     → 확인 → POST /api/fortune/create { type, drawnCards }
     → window.dispatchEvent(byeolkong:balance-updated)
     → router.push(`/fortune/result?id=${data.id}`)
  → 백그라운드 after()가 구조화 리포트 생성 (그동안 generating=true 카드로 /readings·FortuneGeneratingList에 노출)
```

이 뼈대는 기존 사주/궁합 입력 패턴(openConfirm → modal → handleGenerate → create → result)과 동일하며, 카드 뽑기만 그 사이에 끼는 구조다.

### 4.4 입력 화면 컴포넌트

`components/fortune/tarot/TarotInput.tsx` (신규, `CompatInput.tsx` 패턴 참조):
- `type: FortuneType` prop으로 5종 모두 처리
- FORTUNE_CONFIG에서 label/emoji/cost, TAROT_POSITIONS에서 slotLabels 도출
- balance 조회, StarConfirmModal, FortuneRefundModal, INSUFFICIENT_STARS 처리 재사용
- `app/fortune/tarot/[type]/page.tsx`가 이 컴포넌트를 렌더

## 5. 리포트 JSON 구조 & 뷰

### 5.1 JSON 스키마 (도메인 3장)

```jsonc
{
  "headline": "string",          // 한 줄 요약
  "cards": [                      // cardCount 만큼
    {
      "position": "나의 마음",     // 섹션 3 라벨
      "cardName": "The Star",
      "direction": "upright",     // upright | reversed
      "reading": "string"         // 이 포지션에서 이 카드가 말하는 것
    }
  ],
  "summary": "string",           // 카드들을 엮은 종합 내러티브
  "advice": "string"             // 별콩이 마무리 조언
}
```
- **오늘의 타로(1장):** `cards` 길이 1, `summary`/`advice`는 짧게 또는 생략
- 카드 이름/방향은 **뽑기 단계에서 확정** → 프롬프트에 그대로 주입(AI가 카드를 새로 정하지 않음)

### 5.2 저장

`readings` 테이블 재사용:
- `drawn_cards` (JSONB): 뽑은 카드 배열
- 리포트 본문: 기존 fortune이 쓰는 결과 컬럼에 위 JSON 저장
- `emotion_tag`: `fortune:tarot_*` sentinel
- **새 마이그레이션 불필요** (기존 fortune 스키마 재사용). → main-baseline-drift 리스크 회피.

### 5.3 뷰

`components/fortune/TarotReportView.tsx` (신규):
- 상단 headline (별콩이 톤)
- 카드별: 카드 이미지(`public/cards-webp/` 재사용) + 포지션 라벨 + 정/역 + reading
- summary 박스 + advice 마무리
- `/fortune/result`가 `fortuneTypeFromTag(tag)`로 base==="tarot" 판별 → 이 뷰로 분기

## 6. 무료 카운트 (오늘의 타로)

사주 daily와 **동일 메커니즘** (새 정책 없음):
- 상수: `TAROT_DAILY_FREE_LIMIT = 5`, `TAROT_DAILY_PAID_COST = 5` (fortune 쪽)
- 카운트: `readings`에서 `emotion_tag='fortune:tarot_daily'` 행 수
- 무료 잔여 조회: `/api/fortune/tarot-daily-status` (사주 `daily-status` 복사·치환)
- 입력 화면에서 잔여 표시 + 무료면 모달 생략하고 바로 생성

---

## 7. 에러 처리 & create 라우트

기존 `/api/fortune/create`를 확장해 타로 5종 type을 받는다:
- **drawnCards 검증:** 길이 == cardCount, `card_id` 0~77, `direction ∈ {upright, reversed}` (채팅 타로 검증 로직 재사용)
- **잔액 부족:** `INSUFFICIENT_STARS` → 입력 화면 "별 모자라 + 상점 가기" (사주/궁합 동일 UX)
- **생성 실패:** 자동 환불 → `refunded` 플래그 → `FortuneRefundModal` 재사용
- **rate limit:** 기존 create 라우트 정책 그대로
- **무료 오늘의 타로:** 차감 0; 5회 초과 시 5별 차감 경로

## 8. 파일 영향 요약

**신규:**
- `components/tarot/CardDrawRitual.tsx` — 추출된 덱 UI 공유 컴포넌트
- `components/fortune/tarot/TarotInput.tsx` — 운세 타로 입력 화면
- `components/fortune/TarotReportView.tsx` — 타로 리포트 뷰
- `app/fortune/tarot/[type]/page.tsx` — 동적 라우트
- `app/api/fortune/tarot-daily-status/route.ts` — 오늘의 타로 무료 잔여

**수정:**
- `lib/fortune/types.ts` — FortuneType 5종 추가, FORTUNE_CONFIG, MAX_TOKENS_BY_FORTUNE, TAROT_POSITIONS 맵, daily 상수
- `app/tarot/draw/page.tsx` — CardDrawRitual 사용하도록 리팩터 (동작 동일)
- `app/api/fortune/create/route.ts` — 타로 type + drawnCards 처리
- `app/fortune/result/...` — base==="tarot"면 TarotReportView 분기
- AI 프롬프트(fortune 생성 쪽) — 타로 도메인별 리포트 생성 지침 + 카드/포지션 주입

**스키마 변경: 없음** (기존 readings/fortune 재사용).
