# 마이페이지 위계·리듬 개선 (A+C 하이브리드) — 설계

> 작성일 2026-06-22 · 범위: 마이페이지(`app/mypage/page.tsx`) 한정

## 1. 문제 / 배경

현재 마이페이지는 **프로필 카드 · 사주 목록 카드 · 계정 메뉴 카드**가 모두 같은 시각 언어(`cream-warm`/`white` + `rounded-2xl` + `border`)를 써서 위계가 평평하고 단조롭다. 특히:

- 프로필 히어로(이건희 + 명식)와 **사주 목록 첫 행 "이건희(나)"가 같은 정보를 두 번** 노출 → "프로필과 목록이 너무 비슷"한 가장 큰 원인.
- 사주 목록 행이 텍스트 + 연필/쓰레기통 아이콘만 있어 "리스트로서 미완성" 느낌.
- 고객센터/회원탈퇴가 콘텐츠와 같은 흰 카드라 "설정"으로 구분되지 않음.

`내 고민톡(/readings)` 리스트는 이미 리딩 아바타 타일 + 제목 + 칩 + 서브텍스트로 충분히 리치하다 → 그 수준의 완성도를 마이페이지에도 가져온다.

## 2. 방향 결정 (확정)

**A(섹션 위계) 골격 + C(리치 카드) 행** 하이브리드.

| 항목 | 결정 |
|---|---|
| 전체 구조 | 잔액 / 프로필 히어로 / **[내 사주·지인] 섹션** / **[계정] 섹션** / 로그아웃 — 섹션 라벨로 3구역 분리 |
| self 중복 | **사주 목록에서 self(나) 행 제거** — self는 히어로가 전담. 목록은 지인만 |
| 사주 행 | C 스타일 미니 카드: **일주 한자 타일(오행 컬러)** + 이름 + 관계칩 + 일간/생일 서브텍스트 |
| 행 액션 | **케밥(⋯) → 시트**로 수정/삭제 (연필+쓰레기통 상시노출 제거) |
| 계정 메뉴 | "계정" 섹션 라벨 + 설정형 행(좌측 아이콘 + chevron) 그룹. 고객센터 행의 미확인 🔴 점 유지 |
| 로그아웃 | 지금처럼 `bg-eye-purple` 진보라 풀버튼 분리 유지 |
| 프로필 히어로 | `cream-warm` 유지 + 약한 테두리 + `shadow-sm`로 살짝 띄움. 사주 행은 그림자 없음 → 위계 발생 |

## 3. 전체 레이아웃

```
잔액 카드 (보라 그라데이션)              ← 변경 없음
프로필 히어로 (아바타 + 이건희 + 생일 + 명식 4기둥 + 오행)
  · cream-warm + 약한 border + shadow-sm (살짝 띄움)
  · 내부 작은 라벨 "내 명식"
  · 우측 "수정"
[내 사주 · 지인]            + 지인 추가   ← 섹션 라벨 (지인 수만 카운트)
  · 지인 사주 행 (일주 타일)  × N
  · 지인 0명: 빈 상태 카드 ("아직 함께 보는 사주가 없어 · 지인 추가하기")
[계정]                                   ← 섹션 라벨
  · 🎧 고객센터 / 문의      (🔴 미확인)  chevron
  · 👤⊖ 회원 탈퇴                        chevron
로그아웃 (진보라 풀버튼)                 ← 변경 없음
Footer                                   ← 변경 없음
```

기존의 모호한 디바이더(`h-px bg-lilac-mid/30`)는 섹션 라벨 구조로 대체.

## 4. 지인 사주 행 (컴포넌트 단위)

각 행 = 미니 카드 (`cream-warm`, 연한 border, 그림자 없음):

```
┌───────────────────────────────────────────┐
│ [乙巳]  테스트  [친구]                  ⋯  │
│  타일   을목 일간 · 1996.06.02 · 묘시        │
└───────────────────────────────────────────┘
```

- **일주 타일** (좌측, 42×42, `rounded-xl`):
  - 표시: 일주 한자 2자 = `saju.pillars.day.hanja` (예: `乙巳`)
  - 배경색: **일간(일 천간)의 오행 컬러** — `SajuBoard`의 `ELEMENT_COLORS[el].bg` + `text` 재사용 (금=흰색 포함). 일간 오행은 `SajuBoard`의 stem→element 매핑 로직 재사용.
- **이름 + 관계칩**: `display_name` + 관계 라벨 칩(친구/가족/연인/기타). self는 목록에서 제외되므로 "나" 칩은 없음.
- **서브텍스트**: `{일간 오행명} 일간 · {birthDate} · {시진}` (예: "을목 일간 · 1996.06.02 · 묘시"). 시간 모름이면 "· 시간 모름".
- **케밥(⋯)**: 탭 → 하단 시트(또는 인라인 액션) → 수정 / 삭제. 수정은 기존 `ProfileForm` 인라인 편집 흐름 재사용, 삭제는 기존 확인 모달 재사용. **동작·API 변경 없음**, 노출 UI만 케밥으로 정리.
- 데이터: 각 프로필의 `saju: SajuResult`가 이미 로드됨 → 추가 fetch/계산 없음.

### 빈 상태 (지인 0명)
"아직 함께 보는 사주가 없어. 지인을 추가하면 여기에 모아서 함께 풀어볼 수 있어." + "지인 추가하기" 버튼. (기존 문구 재사용)

## 5. 계정 메뉴 + 로그아웃

- "계정" 섹션 라벨 아래 한 컨테이너(`cream-warm`/연한 border), 행 사이 헤어라인:
  - **고객센터 / 문의** — 좌측 인라인 SVG 아이콘(헤드셋/말풍선류, `BottomTab`의 MDI 단일 path 스타일 따름 — Tabler 등 외부 의존성 추가 금지, lilac-deep 톤), 라벨, 우측 chevron. `supportUnread > 0`이면 라벨 옆 🔴(`bg-lilac-deep`) 점 — **지난 사이클 추가분 유지**. `Link href="/mypage/support"`.
  - **회원 탈퇴** — 좌측 인라인 SVG 아이콘(약한 톤), 라벨 `text-light`, chevron. 기존 탈퇴 확인 모달 트리거.
- **로그아웃**: 별도 `bg-eye-purple text-white` 풀버튼 유지(액션 버튼이라 메뉴 리스트와 구분). 동작 변경 없음.

## 6. 프로필 히어로 차별화

- 컨테이너 `cream-warm` 유지 + 테두리 약하게(`border-lilac-mid/20` 수준) + `shadow-sm`(부드러운 그림자)로 살짝 띄움 → "히어로 카드"임을 명확히. 사주 행/메뉴 행은 그림자 없음.
- 내부 좌상단(이름 옆 또는 명식 위)에 작은 라벨 "내 명식".
- 골드/보라 강조는 추가하지 않음 (잔액 카드 보라와 충돌·과함 방지, 차분한 무드 유지).

## 7. 구현 범위 / 비범위

- **범위**: `app/mypage/page.tsx`의 사주 목록 렌더링 + 계정 메뉴 영역 + 프로필 히어로 컨테이너 스타일. 케밥 시트 UI. 사주 행의 일주 타일/서브텍스트 도출.
- 일주 타일/오행 색상은 `components/saju/SajuBoard`의 `ELEMENT_COLORS` + stem→element 매핑을 **재사용**(중복 정의 금지). 필요 시 작은 헬퍼로 추출하되 마이페이지 내부 한정.
- **비범위(이번 사이클 아님)**: 공용 리스트 컴포넌트 추출, `/readings`·`/shop` 등 다른 탭 전파(여기서 검증 후 다음 사이클), 잔액 카드/Footer 변경, 사주 목록의 데이터/API 변경.

## 8. 성공 기준

- 프로필 히어로와 그 아래 영역이 **명확히 다른 시각 무게**를 가진다 (그림자 + 섹션 라벨 + 행 스타일 차이).
- 사주 목록에 self가 더 이상 중복 표시되지 않는다.
- 사주 행이 일주 한자 타일 + 일간 정보로 "완성된 리스트"로 보인다.
- 고객센터/회원탈퇴가 "설정" 영역으로 콘텐츠와 구분된다.
- 로그아웃 버튼·미확인 배지 등 직전 사이클 결정은 보존된다.
- 빌드 통과(`npm run build`) + dev 브라우저에서 위 항목 육안 확인.
