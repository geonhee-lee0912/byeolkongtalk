# 내 고민톡(readings) 표면 통일 리디자인 — 설계

> 작성일 2026-06-22 · 범위: `app/readings/page.tsx` 한정 · 마이페이지 리뉴얼과 톤 통일

## 1. 배경 / 목적

마이페이지를 "흰 카드 + 부드러운 그림자"로 리뉴얼해 베이지 단조로움을 해소했다. 내 고민톡(`/readings`) 리스트는 이미 "리딩 타일 + 제목/서브텍스트/프리뷰" 리치 카드 구조지만, 카드 표면이 여전히 `cream-warm`이라 크림 배경과 거의 같은 톤 → 카드가 안 떠 보인다. 마이페이지와 같은 표면 톤으로 통일한다.

**범위 결정: 표면 통일 위주.** 카드 정보 구조(제목/칩/날짜/프리뷰 배치)나 레이아웃/탭 동작은 건드리지 않는다.

## 2. 변경 사항 (확정)

| 대상 | 현재 | 변경 |
|---|---|---|
| 리스트 카드 (고민 상담 + 운세, 생성중 제외) | `bg-cream-warm` + `border-lilac-mid/30` | **`bg-white` + `border-lilac-mid/20` + `shadow-[0_2px_10px_rgba(159,138,208,0.08)]`** (기존 `hover:border-lilac-deep/50` 유지) |
| 빈 상태 카드 (상담/운세 둘 다) | `bg-cream-warm` + `border-lilac-mid/30` | `bg-white` + `border-lilac-mid/20` + 동일 shadow |
| 탭 활성 상태 | 활성 `bg-cream-warm` | **활성 `bg-white`** (기존 `shadow-sm` 유지). 컨테이너 `bg-lilac-soft/40`는 그대로 |
| 리딩 타일 프레임 | 상담 `w-12 h-12 rounded-xl`, 운세 `w-10 h-10 rounded-lg` (제각각) | **전부 `w-12 h-12 rounded-xl`로 통일** (아이콘/이미지 내용·타일 배경색은 타입별 그대로) |
| 생성중 카드 (다크) | `bg-gradient-to-br from-night to-night-deep` | **변경 없음** (별콩이 생성 무드 유지) |
| 헤더 / 탭 동작 / 페이지네이션 / 카드 정보 구조 | — | **변경 없음** |

## 3. 디테일

### 리스트 카드 표면
- 상담 탭 카드(`Link`)와 운세 탭 카드(`Link`) 양쪽의 컨테이너 className에서 `bg-cream-warm` → `bg-white`, `border-lilac-mid/30` → `border-lilac-mid/20`, `shadow-[0_2px_10px_rgba(159,138,208,0.08)]` 추가. 마이페이지 지인/문의 카드와 동일 shadow 토큰.

### 리딩 타일 통일
- 운세 카드의 아이콘 타일(`w-10 h-10 rounded-lg bg-gold-soft/30`)을 `w-12 h-12 rounded-xl bg-gold-soft/30`로. 아이콘 폰트 크기는 타일에 맞게 소폭 조정(기존 `text-[18px]` 유지 가능).
- 상담(사주 이미지/🔮 fallback)·타로 타일은 이미 `w-12 h-12 rounded-xl` → 변경 없음. 타일 배경색(lilac-soft / cream+border / gold-soft)은 타입 구분용이라 유지.
- 흰 카드 위에서 cream 타일(사주/타로)은 테두리(`border-lilac-soft`)로 윤곽이 유지되므로 가독성 문제 없음.

### 탭 활성색
- `tab === t.key` 분기의 `bg-cream-warm text-eye-purple shadow-sm` → `bg-white text-eye-purple shadow-sm`. 비활성은 `text-text-light/70` 그대로.

### 생성중 다크 카드
- 운세 생성중(`r.generating`) 카드는 `night` 그라데이션 + 골드 라벨 + 스피너 그대로. 표면 통일 대상 아님(의도적 다크).

## 4. 구현 범위 / 비범위

- **범위**: `app/readings/page.tsx`의 카드 표면 className(상담/운세/빈상태), 탭 활성 className, 운세 타일 프레임 크기.
- **비범위**: 카드 안 텍스트/칩/프리뷰 배치 재구성, 탭/필터 동작, 페이지네이션(이미 5개·숫자형 통일됨), 생성중 다크 카드 룩, 헤더 문구, 다른 탭/페이지.

## 5. 성공 기준

- 상담·운세 리스트 카드가 흰색 + 그림자로 크림 배경 위에 또렷이 뜬다 (마이페이지와 동일 톤).
- 활성 탭이 흰색으로 카드와 일관된다.
- 리딩 타일이 모두 동일한 크기·모서리(`w-12 h-12 rounded-xl`)로 정렬된다.
- 생성중 다크 카드는 그대로 유지된다.
- 카드 정보(제목/칩/프리뷰)와 동작은 그대로 — 회귀 없음.
- 빌드 통과 + dev 브라우저에서 두 탭 + 빈상태 + 생성중 카드 육안 확인.
