# 마이페이지(내 정보) 개편 설계

작성일: 2026-06-06
대상 프로젝트: byeolkong_talk (v2)

## 배경

현재 `/mypage`(내 정보)는 계정 허브 역할이지만 정보 구조가 다음과 같이 분산되어 있다.

- 프로필 카드(아바타+닉네임) → 별 잔액 → **내 사주**(전체 `SajuBoard`) → **지인 사주** 목록 → 회원 탈퇴
- 풀이 히스토리 목록은 마이페이지가 아니라 별도 탭 `/readings`("내 고민톡")가 담당. 마이페이지의 `readings` fetch는 "풀이 N회" 카운트 용도로만 쓰임.
- `오행 분포`는 `SajuBoard` 내부(막대 5줄 블록)에 박혀 있고, 이 컴포넌트는 사주 풀이 **결과 화면**과 사주 선택 **picker**에서도 공유된다.
- 결제 내역 화면이 없다(API `/api/payments/list`는 존재). 별 사용 내역 조회 API도 없다(단 `star_transactions` 테이블은 존재).
- 고객센터/문의 진입점이 없다.

이번 개편은 마이페이지를 "계정·프로필 허브"로 재정렬한다.

## 목표 / 비목표

**목표**
- 내 사주 명식을 상단 프로필 카드로 끌어올려 첫 화면에서 바로 보이게 한다.
- 모든 사주(내 사주 + 지인)를 "사주 목록" 한 곳에서 관리한다.
- 오행 분포를 한 줄 형태로 간소화한다.
- 결제/별 내역 화면과 고객센터 진입점을 추가한다.

**비목표**
- 풀이 히스토리 목록 재구현 (이미 `/readings` 탭이 담당).
- 고객센터 실제 채널 연결 (이번엔 링크만 뚫고 타겟은 보류).
- 알림 설정 / 친구 초대 / 공지·이벤트 (이번 범위 외).

## 상세 설계

### 1. 프로필 카드 (명식 통합)
- 기존 아바타+닉네임 행 아래 divider 추가 → 그 아래 `SajuBoard`(4기둥 + 한 줄 오행) 임베드.
- 기존 단독 "내 사주" 섹션은 **삭제**.
- 명식 미입력 상태(self 프로필 없음): "내 사주 입력하기" 버튼 (기존 로직 재사용).
- 카드 우하단 **[수정]** → 기존 `ProfileForm mode="self"` 인라인 편집 (기존 `editingSelf` 상태/`saveSelf` 재사용).

### 2. 오행 한 줄 (`SajuBoard` 공용 변경)
- 기존 5줄 막대 블록(`SajuBoard.tsx` 약 111–162행)을 **한 줄 형태**로 교체.
- 표기: 오행 5개 카운트 칩(`목2 화1 토3 금1 수1`) + ` · 일간 X(목) · 양3 음5`.
- 가장 많은 오행만 볼드/강조.
- **공용 변경**이므로 결과 화면(`saju/result`)과 picker(`ProfilePicker`)의 `SajuBoard`도 동일하게 한 줄로 바뀐다. (의도된 일관성)
- `SajuBoardCompact`(concern/reading용)에는 오행이 없으므로 영향 없음.

### 3. 사주 목록 (구 "지인 사주")
- 섹션 제목 `지인 사주` → **`사주 목록`**.
- 정렬: **내 사주('나' 뱃지)가 첫 행**, 이후 지인.
- 내 사주 행: 수정만 가능, **삭제 버튼 없음**.
- 지인 행: 기존 수정/삭제 CRUD 유지, `+ 지인 추가` 유지.
- **행 탭 → 펼침**: 평소엔 이름 + 관계 뱃지 + 생일·음양만, 탭하면 펼쳐서 `SajuBoardCompact` 명식 + 수정/삭제 버튼 노출. (지인 많아도 스크롤 절약)
- 기존 `profiles` API / `ProfileForm` / 삭제 확인 모달 그대로 재사용.

### 4. 결제 / 별 내역 (신규 `/mypage/payments`)
- 메뉴 카드 "결제 / 별 내역" → 신규 페이지로 이동.
- 두 가지 데이터:
  - 결제(충전) 내역: 기존 `/api/payments/list` 사용.
  - 별 사용/충전 내역: **신규 `GET /api/stars/transactions`** — 로그인 유저의 `star_transactions`를 최신순으로 반환.
- 화면 구성: 상단 요약(총 충전 금액 / 현재 잔액) + 2탭(결제 내역 / 별 내역) 또는 통합 타임라인. 빈 상태 처리 포함.

### 5. 고객센터 / 문의 (진입점만)
- 메뉴 카드 "고객센터 / 문의" 노출.
- 실제 채널(이메일 `mailto:` / 카카오 채널)은 **미정 → 링크 타겟은 placeholder**로 두고 추후 확정. (UI는 완성, 타겟만 TODO)

### 6. 약관 링크 묶음
- 하단에 `이용약관 · 개인정보처리방침 · 환불정책` 텍스트 링크 한 줄 추가 (기존 `/terms` `/privacy` `/refund` 페이지로 연결).

## 신규/변경 API

### `GET /api/stars/transactions` (신규)
- 로그인 유저의 `star_transactions` 최신순(limit 100) 반환.
- 게스트는 빈 배열.
- 반환 필드(안): `id`, `amount`(부호 포함 ±), `type`(충전/사용/환불/조정 등 source), `createdAt`. 실제 컬럼명은 `20260604000000_stars.sql` 스키마 확인 후 매핑.
- service supabase + `getSession()` 패턴은 `/api/payments/list`와 동일.

## 손대는 파일

- `app/mypage/page.tsx` — 재구성 (프로필 카드 명식 통합 / 사주 목록 펼침 / 메뉴 묶음 / 약관 링크)
- `components/saju/SajuBoard.tsx` — 오행 막대 → 한 줄
- `app/mypage/payments/page.tsx` — 신규 (결제 + 별 내역)
- `app/api/stars/transactions/route.ts` — 신규 (별 내역 조회)

## 결정 사항 / 가정

- 사주 목록 = 내 사주 + 지인 모두 (내 사주는 삭제 불가, 수정 가능).
- 프로필 카드 명식 = 전체 4기둥 그리드(`SajuBoard`).
- 오행 분포 = 제거가 아니라 한 줄로 간소화, `SajuBoard` 공용 변경(결과/ picker 포함).
- 결제 + 별 사용 내역 둘 다 제공.
- 사주 목록 행은 탭하면 펼치는 방식.
- 고객센터는 링크만 뚫고 타겟 보류.

## 미해결 / 추후

- 고객센터 실제 연락 채널 확정.
- `star_transactions` 정확한 컬럼/소스 라벨 매핑 (구현 시 스키마 확인).
- 별 내역 화면 레이아웃: 2탭 vs 통합 타임라인 (구현 단계에서 결정).
