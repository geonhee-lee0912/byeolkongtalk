# 웰컴 별 축소 30 → 20 (2026-07-22)

## 결정

`WELCOME_BONUS_STARS` 30 → **20**. 코드 변경은 상수 1줄 — 지급(카카오 콜백 `charge_stars`)과 노출 카피 3곳(WelcomeStarsModal · `/start` 리본 · 홈 배너)이 모두 이 상수를 참조하므로 일괄 반영된다. 기존 유저 잔액은 불변(신규 가입부터 적용). 광고 소재에는 "30별" 숫자가 박혀 있지 않음(사용자 확인) — 소재 교체 불필요.

## 근거 (2026-07-22 prod 분석, 지인 6명 제외, 웰컴 수령 411명)

**결제자는 맛보기를 거치지 않는다.**
- 결제자 32명 전원 가입 24h 내 결제, 중앙값 **6분**
- 20/32명은 결제 직전 잔액 ≥30(무료별 거의 그대로), 18/32명은 **무료별 소비 0에서 결제**
- 결제 전 평균 소비 8.4별 → 웰컴 별은 "체험 → 신뢰 → 결제" 장치가 아니라 **첫 프리미엄 결제(관계 스프레드 40별)의 할인 쿠폰**으로 기능

**결제의 실체 = 갭 결제, 그리고 ₩1,000는 one-and-done.**
- 21/32명: star_10(₩1,000) 구매 → 즉시 타로 상담 (웰컴 30 + 10 = 관계 스프레드 40)
- star_10 첫 결제자 25명 중 재구매 **2명(8%)**, 평균 결제 1.08회 → 세그먼트 생애 매출 ≈ ₩1,080/인

**비결제자는 소진하고 떠난다.**
- 별 쓴 비결제자 326명 중 71% 소진(잔액<10), 평균 23.3별(상담 1~2회) 소비 = 순수 API 원가

## 기대 효과

1. **첫 결제 단가 상향(주 효과)**: 갭 40−20=20 → star_10으로 못 채움 → **star_30(₩2,800)**이 최소 갭 패키지. 손익분기 = 갭결제자 유지율 36% (₩2,800/₩1,000). 절반이 포기해도 해당 세그먼트 매출 +40%.
2. **비결제 COGS 절감(부 효과)**: 무료 소비 캡 23.3 → 20별.
3. **맛보기 보존**: 20별 = 타로 two_card(15) 또는 이번 달 운세(20) 1회 — "한 번은 제대로"가 유지되는 하한. (15까지 내리면 two_card 단일 선택지)

## 리스크와 롤백

- **₩2,800 허들에서 임펄스 소멸**: 판정 지표 1 하회 시 상수 롤백(1줄).
- **가입 CVR 하락**: 웰컴 20도 랜딩/모달에서 동일하게 노출 — 광고 클릭→가입 전환을 가드레일로 관찰.
- **웰컴 정확 일치 함정 재발(알려진 편차)**: 2026-07-17 스펙 원칙 "유료 상품 가격 = 웰컴 별 정확 일치 금지"에 대해, 이번 달 운세(20별)·구 사주 상담(20별, 신규 진입 폐쇄)이 새 웰컴과 정확 일치하게 됨. 지금은 가격 동시 변경으로 실험을 흐리지 않기 위해 **변경하지 않고 관찰** — fortune_monthly 무료 소진 쏠림이 보이면 후속 조정.
